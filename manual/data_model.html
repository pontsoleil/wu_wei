WuWei データモデル

1. 概要

この文書は WuWei app2 が実行時に扱う新データモデルを定義する。

旧バージョンの Note データは、wuwei.note が読み込む時点でこのモデルへ変換する。 変換後の実行時データには、旧モデル由来の属性を残してはならない。 旧モデル属性が残っている場合、またはこの文書と異なる階層に属性が存在する場合は、互換処理を行わずエラーとして停止する。

モデルの基本方針は次のとおりである。

  • Note は resources[] と pages[] を持つ。

  • Page は nodes、links、groups を持つ。

  • Content ノードは Resource 本体を直接保持せず、resourceRef で参照する。

  • 同じ Resource を複数の Content から参照できる。

  • 同一 Resource の参照ページ、シート、時間範囲などの違いは Content.resourceView に保持する。

  • アップロード Resource の実体は Note 本体とは別に管理し、Note 保存時に Note スナップショットへコピーできる。

  • 外部 Web Resource は原則として実体コピーせず、メタデータと埋め込み情報を保持する。

  • Node の name は廃止し、label を使用する。

  • 表示状態は visible で表す。hidden は使用しない。

  • Link の矢印は routing.startArrow と routing.endArrow で表す。

  • Group 種別は simple、horizontal、vertical、timeline とする。

  • Group の表示/非表示は、Group 全体を一括で制御する。

2. 識別子

内部 ID は UUID v4 に _ を付けた文字列とする。

"_" + uuid.v4()

例:

"_2f754971-e1b3-436c-9148-29c4a18a100f"

ユーザー ID は user_id として保存する。 次の旧表現は実行時データで使用しない。

  • userId

  • userID

  • owner_id

  • ownerId

3. 共通型

3.1. 監査情報

作成者、更新者、作成日時、更新日時を保持する。

{
  "owner": "sambuichi",
  "createdBy": "_2f754971-e1b3-436c-9148-29c4a18a100f",
  "createdAt": "2026-04-26T12:00:00+09:00",
  "lastModifiedBy": "",
  "lastModifiedAt": ""
}
項目 説明

owner

表示用の所有者名。

createdBy

作成者の user_id。

createdAt

作成日時。ISO 8601 形式。

lastModifiedBy

最終更新者の user_id。

lastModifiedAt

最終更新日時。ISO 8601 形式。

3.2. リッチテキスト

Memo、Content、Page、Link などの説明文を表す。

{
  "format": "asciidoc",
  "body": ""
}
項目 説明

format

本文形式。plain/text、asciidoc、markdown、html など。

body

本文。

plain/text はプレーンテキストとして表示する。 asciidoc や markdown は HTML に変換して Memo や info ペインに表示できる。

3.3. 変換情報

Page や描画領域の移動、拡大率を表す。

{
  "x": 0,
  "y": 0,
  "scale": 1
}

3.4. 文字スタイル

文字スタイルを表す。

{
  "family": "sans-serif",
  "size": 14,
  "color": "#000000",
  "align": "center"
}

align は left、center、right のいずれかとする。

3.5. 線スタイル

線スタイルを表す。

{
  "kind": "SOLID",
  "color": "#666666",
  "width": 1
}

kind は SOLID、DOTTED、DASHED、LONG_DASHED のいずれかとする。

3.6. 矢印スタイル

Link の始点または終点に付く矢印を表す。

{
  "kind": "ARROW",
  "size": 12
}

kind は ARROW、HOLLOW-ARROW、LINE-ARROW、RHOMBUS、HOLLOW-RHOMBUS、CIRCLE、HOLLOW-CIRCLE のいずれかとする。

3.7. Node スタイル

Node の表示スタイルを表す。

{
  "fill": "#FEFEFE",
  "font": {
    "family": "sans-serif",
    "size": 14,
    "color": "#000000",
    "align": "center"
  },
  "line": {
    "kind": "SOLID",
    "color": "#666666",
    "width": 1
  }
}

4. Note

Note はページ、Resource 定義、監査情報を持つ。 Note 単体で受け渡しても再現できるように、参照 Resource の定義コピーを resources[] に保持する。

{
  "note_id": "_note_uuid",
  "note_name": "WuWei note",
  "description": "",
  "currentPage": "_page_uuid",
  "resources": [],
  "pages": [],
  "audit": {}
}
項目 説明

note_id

Note ID。

note_name

Note 名。

description

Note の説明。

currentPage

現在表示している Page ID。旧設計のページ番号ではなく、_page_uuid を保存する。

resources

この Note が参照する Resource 定義のコピー。

pages

表示順に並ぶ Page 配列。各 Page は恒久 ID として _page_uuid を持つ。

audit

監査情報。

resources[] は単なるインデックスではない。 Note の再現に必要な Resource メタデータと保存情報を保持する。 Content ノードは resourceRef により resources[].id を参照する。

5. Page

Page は Node、Link、Group を保持する。 Page にはページ番号を恒久 ID として持たせない。 Note の pages[] は表示順に並ぶ配列であり、各 Page の id は _page_uuid とする。 ページ番号は pages[] 内の順序から計算される表示用番号である。 ページが追加または削除されると、同じ _page_uuid の Page でもページ番号は変わり得る。 ページ一覧や draw 上の表示では、この順序番号をページ番号として使用する。

{
  "id": "_page_uuid",
  "label": "Page label",
  "description": {
    "format": "plain/text",
    "body": ""
  },
  "nodes": {},
  "links": {},
  "groups": {},
  "transform": {
    "x": 0,
    "y": 0,
    "scale": 1
  },
  "thumbnail": null,
  "audit": {}
}

nodes、links、groups はそれぞれ ID をキーとするオブジェクトとする。 配列として保存しない。 旧設計の pp は保存しない。 ページ番号が必要な場合は、pages[] の配列 index から 1 始まりで算出する。

6. Resource

6.1. Resource の役割

Resource は Content ノードから参照される再利用可能なコンテンツ定義である。 Resource は大きく、他者の公開情報と自己登録情報に分ける。

分類 説明

外部 Web Resource

Blog、YouTube、Vimeo、組織サイト、公開 PDF URL など。他者の公開情報を参照する。

Uploaded Resource

PDF、Office、画像、動画、音声、テキストなど、ユーザーがアップロードした自己登録情報。

コンテンツの種別と作成経路は別の概念である。 たとえば PDF は外部 Web URL として参照される場合も、アップロードファイルとして登録される場合もある。

6.2. Resource 定義

{
  "id": "_resource_uuid",
  "schemaVersion": 1,
  "origin": "uploaded",
  "identity": {
    "uri": "",
    "canonicalUri": "upload://2026/04/26/document.pdf",
    "title": "document.pdf"
  },
  "media": {
    "kind": "document",
    "mimeType": "application/pdf"
  },
  "viewer": {
    "supportedModes": [
      "infoPane",
      "newTab",
      "newWindow",
      "download"
    ],
    "defaultMode": "infoPane",
    "embed": {
      "uri": "",
      "html": ""
    }
  },
  "storage": {
    "managed": true,
    "copyPolicy": "snapshot",
    "sourcePath": "2026/04/26/document.pdf",
    "primaryPath": "data/{user_uuid}/resource/2026/04/26/{resource_uuid}",
    "snapshotPath": "",
    "files": [
      {
        "role": "original",
        "path": "2026/04/26/document.pdf",
        "sourcePath": "C:/Apache24/htdocs/wu_wei2/data/{user_uuid}/upload/2026/04/26/document.pdf",
        "mimeType": "application/pdf",
        "size": 123456,
        "sha256": "..."
      },
      {
        "role": "thumbnail",
        "path": "thumbnail.png",
        "mimeType": "image/png",
        "size": 12345,
        "sha256": "..."
      },
      {
        "role": "preview",
        "path": "preview.pdf",
        "mimeType": "application/pdf",
        "size": 123456,
        "sha256": "..."
      }
    ]
  },
  "rights": {
    "owner": "_user_uuid",
    "copyright": "",
    "license": "",
    "attribution": ""
  },
  "audit": {}
}

6.3. 由来

origin は Resource の由来を表す。

値 説明

web

外部 Web URL。Blog、YouTube、Vimeo、組織サイトなど。

uploaded

ユーザーがアップロードしたファイル。

generated

WuWei が生成した派生 Resource。

6.4. メディア情報

media.kind は Resource の表示分類を表す。 YouTube や Vimeo の URL は webpage ではなく video として扱う。

値 例

webpage

通常の Web ページ。

video

YouTube、Vimeo、アップロード動画。

audio

アップロード音声。

document

PDF、Office、テキスト文書。

image

写真、画像。

general

特定のビューアを持たない一般 Resource。

media.mimeType は MIME type を保持する。

6.5. 表示情報

viewer は Resource の表示方法を定義する。

項目 説明

supportedModes

利用可能な表示方法。infoPane、newTab、newWindow、download など。

defaultMode

標準の表示方法。

embed.uri

iframe などで埋め込む URL。

embed.html

必要に応じて保存する埋め込み HTML。

6.6. 保存情報

storage は Resource の保存場所とコピー方針を定義する。

項目 説明

managed

WuWei がローカルファイルを管理する場合は true。

copyPolicy

snapshot または metadataOnly。

sourcePath

アップロード元ファイルの相対パス。例: 2026/04/26/document.pdf。

primaryPath

Resource ライブラリのディレクトリ。

snapshotPath

Note スナップショット内の Resource ディレクトリ。Note 保存時に設定される。

files

Resource に関連するファイル一覧。

アップロード元ファイルの実体は次の場所に保存する。

data/{user_uuid}/upload/yyyy/mm/dd/{filename}

Resource ライブラリでは、アップロード元ファイルを重複保存しない。 storage.files[].path に upload 領域からの相対パスを保持する。

Resource ライブラリの保存例:

data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/resource.json
data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/thumbnail.*
data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/preview.pdf

Note スナップショットの保存例:

data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/resource.json
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/original.*
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/thumbnail.*
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/preview.pdf

6.7. Resource の重複判定

Resource ライブラリでは、同じ実体を重複保存しない。 異なる Note や異なる content_uuid から参照されても、同じ Resource と判定できる場合は同じ resource_uuid を使う。

重複判定は次の規則に従う。

  • storage.sourcePath が同じ yyyy/mm/dd/filename であれば、sha256 が異なっていても同じアップロードファイルの更新とみなし、Resource を増やさない。

  • sha256 が同じであれば、参照 Content が異なっていても同じ実体とみなし、既存 Resource を再利用する。

  • storage.sourcePath と sha256 の両方が異なる場合は、新しい Resource として作成できる。

  • Note スナップショットは Note ごとに保持してよく、Note 間で重複排除しない。

7. Content と ResourceView

resourceView は Content ノードごとに定義する。 同じ Resource のどのページ、シート、時間、断片を参照するかを表す。

{
  "mode": "documentPage",
  "page": 12,
  "fragment": "#page=12",
  "label": "p.12"
}
項目 説明

mode

default、documentPage、documentRange、spreadsheetSheet、mediaTime、mediaRange、webFragment など。

page

PDF や文書プレビューの単一ページ番号。

startPage

文書範囲の開始ページ。

endPage

文書範囲の終了ページ。

sheet

Spreadsheet のシート名。

cell

Spreadsheet のセルまたは範囲。

time

Media の開始時刻。秒単位。

start

Media 範囲の開始時刻。秒単位。

end

Media 範囲の終了時刻。秒単位。

fragment

URI fragment 表現。

label

参照位置の表示ラベル。

  "description": {
    "format": "plain/text",

  },

8.3. Content ノード

Content は Resource を参照する Node である。

{
  "id": "_content_uuid",
  "type": "Content",

9. Link

Link は Node 間の接続を表す。

{
  "id": "_link_uuid",
  "type": "Link",
  "from": "_source_node_uuid",
  "to": "_target_node_uuid",
  "x": 0,
  "y": 0,
  "x2": 0,
  "y2": 0,
  "shape": "NORMAL",
  "visible": true,
  "relation": "",
  "label": "",
  "description": {
    "format": "plain/text",
    "body": ""
  },
  "style": {
    "font": {
      "family": "sans-serif",
      "size": 14,
      "color": "#000000",
      "align": "center"
    },
    "line": {
      "kind": "SOLID",
      "color": "#666666",
      "width": 2
    }
  },
  "routing": {
    "path": "",
    "startArrow": null,
    "endArrow": {
      "kind": "ARROW",
      "size": 12
    }
  },
  "audit": {}
}

shape は NORMAL、HORIZONTAL、HORIZONTAL2、VERTICAL、VERTICAL2 などを使用できる。 x と y は Link 形状を編集するための第 1 制御点である。 HORIZONTAL2 と VERTICAL2 は 2 つの制御点を持つため、x2 と y2 を第 2 制御点として保存する。 NORMAL、HORIZONTAL、VERTICAL では x2 と y2 は使用しない。 曲線や折れ線の追加情報は必要に応じて routing に保存する。

shape 制御点

NORMAL

マウスとリンク線分の交点を一時的な編集点として扱う。固定制御点は通常持たない。

HORIZONTAL

x,y を第 1 制御点として使用する。

VERTICAL

x,y を第 1 制御点として使用する。

HORIZONTAL2

x,y を第 1 制御点、x2,y2 を第 2 制御点として使用する。

VERTICAL2

x,y を第 1 制御点、x2,y2 を第 2 制御点として使用する。

10. Group

Group は複数の Node、Link、または Group 軸をまとめて扱うための定義である。

{
  "id": "_group_uuid",
  "type": "simple",
  "visible": true,
  "members": [
    {
      "id": "_node_uuid",
      "role": "member"
    }
  ],
  "spine": {},
  "style": {},
  "audit": {}
}

Group 種別は次のいずれかとする。

  • simple

  • horizontal

  • vertical

  • timeline

Group の表示/非表示は Group 全体に対して適用する。 Group のメンバー Node のいずれかを Group 操作として非表示にする場合、その Group 全体を非表示にする。 同じ規則は timeline Group にも適用する。

10.1. simple Group

simple Group はメンバー全体を囲むグループである。 囲み線はメンバーの現在位置に追従して描画する。

10.2. horizontal / vertical Group

horizontal Group と vertical Group は軸を持つグループである。 軸は表示対象であり、ドラッグ操作によって移動できる。 simulation モードでは、Group の形状を不用意に変形しない。

10.3. timeline Group

timeline Group は時間軸を持つグループである。 spine.axis に開始時刻、終了時刻、軸方向、表示位置を保持する。

12. 2026-04-27 補足: Resource URI と info 表示

この章は、wuwei.home、wuwei.note、wuwei.info の現行実装に合わせて、Resource の URI 解決と info ペイン表示の前提を明確にする。

12.1. Content から Resource への参照

保存済み Note の Content ノードは、Resource 本体を node.resource として保持しない。 Content ノードは必ず resourceRef で Note 本体の resources[] 内の Resource を参照する。 読込時には wuwei.note が resourceRef と resources[].id を照合し、実行時表示用に node.resource を復元する。

実行時の node.resource は描画・編集・info 表示のための一時展開であり、保存時には再び resourceRef に戻す。 保存データに node.resource、node.media、node.uri、node.url を残してはならない。

12.2. Resource URI の解決順序

info ペイン、外部ウィンドウ表示、Resource 追加処理で表示 URL を求めるときは、次の順序で URI を解決する。

resource.identity.uri
resource.viewer.embed.uri
resource.snapshotSources.previewUri
resource.identity.canonicalUri
resource.snapshotSources.originalUri

identity.uri は Resource の通常表示 URL である。 YouTube、Vimeo など、通常 URL と iframe 埋め込み URL が異なる場合は、通常 URL を identity.uri、iframe 用 URL を viewer.embed.uri に保持する。 PDF や Office 文書の preview がある場合は、表示用 preview を snapshotSources.previewUri または storage.files[].role = "preview" から導出できるようにする。

URI が完全に空の Resource は、Resource としては存在しても info ペインに表示対象を持たない。 この場合、表示側は URL を推測してはならない。 ただし、既存 Resource に URI がある状態で Note 保存や Resource 更新を行うとき、空 URI の更新データで既存 URI を上書きしてはならない。

12.3. iframe 表示

info ペインで Web ページ、YouTube、Vimeo などを表示する場合、PHP proxy は使用しない。 server/proxy.php などの PHP スクリプトを iframe の src に設定してはならない。 サーバー CGI は shell script を基本とし、PHP 実行環境を前提にしない。

Web ページは原則として URI を直接 iframe に設定する。 対象サイトが X-Frame-Options や CSP により iframe 表示を拒否する場合、info ペインは iframe を隠し、URL と「iframe 表示を拒否された」旨のメッセージを表示する。 この場合でも、新しいタブまたは別ウィンドウで開くためのリンクは提供する。

YouTube と Vimeo は media.kind = "video" として扱う。 通常の Web ページを意味する webpage には分類しない。 埋め込み URL は viewer.embed.uri に保持する。

12.4. URI を持つ Resource の例

{
  "id": "_resource_uuid",
  "type": "Resource",
  "identity": {
    "title": "XBRL Global Ledger Taxonomy Framework 2017",
    "uri": "https://www.xbrl.org/int/gl/2016-12-01/gl-framework-2017-PWD-2016-12-01.html",
    "canonicalUri": "https://www.xbrl.org/int/gl/2016-12-01/gl-framework-2017-PWD-2016-12-01.html"
  },
  "media": {
    "kind": "webpage",
    "mimeType": "text/html",
    "downloadable": false
  },
  "viewer": {
    "supportedModes": ["infoPane", "newTab", "newWindow"],
    "defaultMode": "infoPane"
  },
  "storage": {
    "managed": false,
    "copyPolicy": "metadataOnly",
    "primaryPath": "data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}",
    "snapshotPath": "",
    "files": []
  },
  "snapshotSources": {},
  "rights": {},
  "audit": {}
}

12.5. iframe 用 URI を持つ動画 Resource の例

{
  "id": "_resource_uuid",
  "type": "Resource",
  "identity": {
    "title": "小栗判官（7段目・照手車引き）",
    "uri": "https://vimeo.com/933802755",
    "canonicalUri": "https://vimeo.com/933802755"
  },
  "media": {
    "kind": "video",
    "mimeType": "text/html",
    "downloadable": false
  },
  "viewer": {
    "supportedModes": ["infoPane", "newTab", "newWindow"],
    "defaultMode": "infoPane",
    "embed": {
      "uri": "https://player.vimeo.com/video/933802755",
      "html": ""
    }
  },
  "storage": {
    "managed": false,
    "copyPolicy": "metadataOnly",
    "primaryPath": "data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}",
    "snapshotPath": "",
    "files": []
  },
  "snapshotSources": {},
  "rights": {},
  "audit": {}
}
