== 目的

本資料は、WuWei の Content node について、`resource.kind` 別に、編集ペイン、表示ペイン、管理者用ペインで扱う項目を整理する。

本版では、次の設計方針を反映する。

* `resource.contents` は新規データでは使用せず、`resource.viewpoint` を使用する。
* 旧 Contents group は廃止し、Viewpoint group として扱う。
* `resource.title`、`resource.uri`、`resource.canonicalUri` は新規データでは使用しない。
* Content の表示名は `node.label` に一本化する。
* remote Content の参照元は `resource.original.url`、正規 URL は `resource.original.canonicalUrl` に保存する。
* upload Content の原本は `resource.original.storageRole` と `resource.storage.files[]` により管理する。
* `graph` 欄とは別に `svg` 欄を設け、`graph.node` / `graph.link` と、`g.node` / `g.link` / SVG DOM の対応を区別する。
* 管理者用ペインは read only の診断ペインとし、管理者ロールの場合だけ表示する。

各表では、次の列を用いる。

[cols="2,1,2,2,4",options="header"]
|===
|項目名 |更新可 / 参照 |graph |svg |内容

|項目名
|更新可又は参照
|`graph.node` 又は `graph.link` の対応属性
|`g.node`、`g.link`、`text.node-label`、`foreignObject` などの SVG DOM 対応
|項目の意味
|===

== 基本方針

=== current.page / graph / svg の三層

WuWei では、保存正本、画面操作用 runtime model、SVG DOM を区別する。

[cols="20%,30%,50%",options="header"]
|===
|層 |役割 |主な内容

|`current.page`
|保存正本
|保存対象となる `nodes[]`、`links[]`、`groups[]` を保持する。

|`graph`
|画面操作用 runtime model
|SVG 描画、選択、ドラッグ、hover、context menu、編集操作の作業対象である。

|SVG DOM
|実際の描画要素
|D3 により生成される `g.node`、`g.link`、`rect.shape-node`、`text.node-label`、`foreignObject`、`path`、`image` などである。
|===

=== current.page と graph の関係

現行実装では、ページ読込時又は graph 再構成時に、`current.page.nodes[]`、`current.page.links[]`、`current.page.groups[]` から `graph.nodes[]`、`graph.links[]`、`graph.groups[]` を構成する。

[source,text]
----
current.page.nodes[]  ─┐
current.page.links[]  ─┼─> graph.nodes[] / graph.links[] / graph.groups[]
current.page.groups[] ─┘
----

現在の実装では、可読性及び保守性を優先し、`graph.nodes[]` は `current.page.nodes[]` の node object を参照する構成を基本とする。したがって、通常 node については、`graph.node` と `current.page.nodes[]` の対応 node が同一 object である場合がある。

この場合、操作中は `graph` を作業対象として扱うが、保存 JSON は `current.page` から生成する。`graph` から保存データを直接生成しない。

NOTE: `forward` / `backward` による重なり順変更は、`graph.nodes[]` の表示順を一時的に変更する操作であり、`current.page.nodes[]` の保存順には反映しない。

=== graph に含まれる詳細属性の扱い

現行実装では、`graph.node` に `resource.original`、`resource.storage.files[]`、`resource.viewer`、`resource.media`、`resource.rights.*`、`audit` などの詳細属性が含まれる場合がある。

ただし、これらは SVG 描画そのものに必要な属性ではない。SVG 描画で直接使用する主な属性は、`id`、`type`、`x`、`y`、`visible`、`state`、`label`、`description`、`shape`、`size`、`style`、`thumbnailUrl`、`iconKind` などである。

将来、SVG 描画性能が問題となる場合は、`graph.node` を軽量化し、詳細属性を `current.page.nodes[]` 側のみで保持する設計へ移行できる。ただし、本版では現行実装との整合を優先し、`graph.node` に詳細属性が存在する可能性を許容する。

=== graph と SVG DOM の分離

`graph.node` 及び `graph.link` は、SVG 描画と操作に使う runtime model である。

`g.node` 及び `g.link` は、D3 により SVG 上に生成又は更新される DOM 要素である。

本資料では、`graph` 欄に runtime model の対応属性を記載し、`svg` 欄に SVG DOM 又は D3 selection の対応を記載する。

[cols="1,2,3",options="header"]
|===
|区分 |例 |意味

|保存正本
|`current.page.nodes[]`, `current.page.links[]`, `current.page.groups[]`
|保存対象となる正本データである。

|graph runtime
|`graph.node`, `graph.link`
|SVG 描画、選択、ドラッグ、簡易表示に用いる runtime model である。

|SVG DOM / D3 selection
|`g.node`, `g.link`, `rect.shape-node`, `text.node-label`, `foreignObject`
|ブラウザ上で実際に描画される SVG 要素である。
|===

=== 通常 link と pseudo link

通常 link は `current.page.links[]` に保存される。

一方、group axis、group outline、timeline axis、viewpoint axis / boundary などは、保存対象 link ではなく、`current.page.groups[]` から `graph.links[]` に展開される pseudo link として扱う。

[cols="25%,25%,25%,25%",options="header"]
|===
|対象 |current.page |graph |svg

|通常 link
|`current.page.links[]`
|`graph.links[]`
|`g.link`

|group axis
|`current.page.groups[]`
|pseudo `graph.link`
|`g.link` 又は専用 SVG 要素

|group outline / perimeter
|`current.page.groups[]`
|pseudo `graph.link`
|`g.link` 又は専用 SVG 要素

|timeline axis
|`current.page.groups[]`
|pseudo `graph.link`
|timeline axis SVG 要素

|viewpoint axis / boundary
|`current.page.groups[]`
|pseudo `graph.link`
|viewpoint axis / boundary SVG 要素
|===

pseudo link は `current.page.links[]` に保存しない。context menu で pseudo link を target として受け取った場合は、`target.groupRef` から `current.page.groups[]` の group を解決して操作する。

=== label と description.body

`label` は Node の短い表示名である。

`description.body` は保存正本の本文又は説明文である。

`description.format` は、`description.body` の記法を示す。値は、例えば `plain`、`asciidoc`、`markdown` などである。

現在の SVG 描画では、Memo 及び Content の `description.body` は `description.format` に従って HTML 化され、SVG の `foreignObject` 内に描画される。

[cols="1,1,2,2,4",options="header"]
|===
|対象 |表示元 |graph |svg |内容

|Topic
|`label`
|`graph.node.label`
|`g.node text.node-label`
|概念名を表示する。`description.body` は説明であり、通常は図形内本文としては使用しない。

|Memo
|`description.body`
|`graph.node.description.body`
|`g.node foreignObject`
|Memo は `label` を持たない。`description.body` を HTML 化して図形内に表示する。

|Content
|`label` 及び `description.body`
|`graph.node.label`, `graph.node.description.body`
|`g.node text.node-label`, `g.node foreignObject`
|Content の見出しは `label`、図形内説明文は `description.body` である。

|Segment
|`label`
|`graph.node.label`
|`g.node text.node-label`
|Timeline Segment の表示名を表示する。

|PageMarker
|`label`
|`graph.node.label`
|`g.node text.node-label`
|Viewpoint group の PageMarker の表示名を表示する。
|===

NOTE: `graph.node.text` を `description.body` の正本として扱わない。保存正本は `current.page.nodes[].description.body` である。

=== 廃止項目

新規データでは、次の項目を使用しない。

[cols="1,3",options="header"]
|===
|項目 |扱い

|`resource.title`
|定義しない。Content の表示名は `node.label` に一本化する。

|`resource.uri`
|定義しない。remote Content の参照 URL は `resource.original.url`、upload Content の原本は `resource.original.storageRole` と `resource.storage.files[]` により管理する。

|`resource.canonicalUri`
|定義しない。remote Content の正規 URL は `resource.original.canonicalUrl` に保存する。

|`resource.contents`
|新規データでは使用しない。旧 v2 データの legacy 項目としてのみ扱い、読み込み時に `resource.viewpoint` へ移行する。
|===

=== resource.original

`resource.original` は、Content の原本又は参照元を表す。

==== upload Content の original

upload Content の場合、`resource.original` は登録済み原本ファイルを示す。

実際のファイル情報は `resource.storage.files[]` に保存する。

[source,javascript]
----
resource: {
  source: "upload",
  kind: "document",
  documentKind: "pdf",

  original: {
    type: "upload",
    storageRole: "original"
  },

  storage: {
    files: [
      {
        role: "original",
        path: "2026/05/24/file_uuid/report.pdf",
        dir_name: "data/user_uuid/upload/2026/05/24/file_uuid/",
        file_name: "report.pdf",
        mimeType: "application/pdf",
        size: 123456,
        sha256: "..."
      },
      {
        role: "thumbnail",
        path: "2026/05/24/file_uuid/thumbnail.jpg",
        file_name: "thumbnail.jpg",
        mimeType: "image/jpeg"
      },
      {
        role: "pdf-preview",
        path: "2026/05/24/file_uuid/preview.pdf",
        file_name: "preview.pdf",
        mimeType: "application/pdf"
      }
    ]
  }
}
----

==== remote Content の original

remote Content の場合、`resource.original` は外部資料の参照情報を示す。

登録時又は現在参照する URL は `resource.original.url` に保存する。

外部資料の正規 URL は `resource.original.canonicalUrl` に保存する。

これらは Content 定義時の入力項目であり、remote Content では編集可能とする。

[source,javascript]
----
resource: {
  source: "remote",
  kind: "html",

  original: {
    type: "remote",
    url: "https://example.com/article?id=123&utm_source=mail",
    canonicalUrl: "https://example.com/article?id=123",
    accessedAt: "2026-05-24T10:00:00Z",
    identifiers: [
      { scheme: "doi", value: "10.xxxx/yyyy" },
      { scheme: "ndl", value: "..." }
    ]
  }
}
----

=== resource.viewpoint と legacy resource.contents

新規データでは `resource.contents` は使用しない。

Document / HTML Content のページ番号、anchor、viewer page 変換、PageMarker / Viewpoint に関係する metadata は `resource.viewpoint` に保存する。

旧 v2 以前のデータに `resource.contents` が存在する場合は、読み込み時又は v2 正規化時に `resource.viewpoint` へ移行し、保存時には `resource.contents` を出力しない。

[source,javascript]
----
if (resource.contents && !resource.viewpoint) {
  resource.viewpoint = resource.contents;
}
delete resource.contents;
----

PDF / Office 文書の例を示す。

[source,javascript]
----
resource: {
  source: "upload",
  kind: "document",
  documentKind: "pdf",

  media: {
    pageCount: 12
  },

  viewpoint: {
    firstPageNumber: 1,
    pageOffset: 0,
    pageCount: 12
  }
}
----

HTML の例を示す。

[source,javascript]
----
resource: {
  source: "remote",
  kind: "html",

  viewpoint: {
    anchorIndex: [
      { id: "_introduction", label: "Introduction" },
      { id: "_background", label: "Background" }
    ]
  }
}
----

== 保存時の検査

remote Content の `resource.original.url` 及び `resource.original.canonicalUrl` は、定義時入力項目であるため更新可とする。

ただし、URL 変更により viewer 解決、thumbnail 取得、PageMarker anchor、Segment、外部埋め込み可否が変わる場合がある。

保存時には次の再検査を行う。

[cols="1,3",options="header"]
|===
|検査項目 |内容

|viewer 再解決
|iframe、external、PDF.js、Office Viewer、video player 等の表示方法を再判定する。

|thumbnail 再取得
|必要に応じて remote thumbnail を再取得又は再生成する。

|anchor 再検査
|HTML の場合、Viewpoint group の PageMarker の `anchorHref` と新しい URL の対応を確認する。

|Segment 再検査
|video / audio の場合、duration が変化したときに Segment の範囲を確認する。

|legacy 項目削除
|保存時には `resource.contents`、`resource.uri`、`resource.canonicalUri`、`resource.title` を出力しない。

|警告表示
|URL 変更により既存 PageMarker、Segment、thumbnail が不整合になる可能性を表示する。
|===

upload Content の原本ファイルは、`resource.storage.files[role="original"]` により管理する。

upload Content では、原本ファイルの論理パスを編集ペインで変更しない。

原本を変更する必要がある場合は、URL や path の直接編集ではなく、Content の再登録又は置換操作として扱う。

== まとめ

本仕様では、Content の表示名を `node.label` に一本化し、`resource.title` は定義しない。

Content の参照元は、upload では `resource.original.storageRole` と `resource.storage.files[]`、remote では `resource.original.url` と `resource.original.canonicalUrl` により管理する。

Document / HTML のページ、anchor、viewer page 変換、PageMarker / Viewpoint 関連 metadata は `resource.viewpoint` に保存する。`resource.contents` は旧 v2 互換の legacy 入力としてのみ扱い、新規保存データには出力しない。

`graph` は画面操作用 runtime model であり、`svg` は実際の SVG DOM である。現行実装では `graph.node` に詳細属性が含まれる場合があるが、SVG 描画で直接使用する属性と、編集ペイン・表示ペイン・管理者用ペインで参照する詳細属性を区別して扱う。
