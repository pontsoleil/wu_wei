== 目的

本資料は、WuWei の Content node について、`resource.kind` 別に次の三種類のペインで表示又は編集する項目を整理する。

* 編集ペイン
* 表示ペイン
* 管理者用ペイン

各表では、次の列を用いる。

[cols="3,1,3,3,4",options="header"]
|===
^|項目名 ^|更新 ^|graph ^|svg ^|内容

|項目名
|更新可又は参照
|`graph.node` 又は `graph.link` の対応属性
|`g.node` 又は `g.link` などの SVG DOM 対応
|項目の意味
|===

== 基本方針

=== current.page と graph の役割分担

`current.page` は保存正本である。

`current.page.nodes[]` は、Node の意味属性、表示属性、Content resource、監査情報、PageMarker、Segment などを保持する。

`graph.nodes[]` は SVG 描画用の runtime view model である。

現在の実装では、既存処理との互換性のため `graph.node` が `current.page.nodes[]` の属性を多く参照できる状態にある。ただし、今後の描画性能改善では、`graph.node` に保持する情報を描画に必要な軽量項目へ絞り、編集ペイン、表示ペイン、管理者用ペインは `current.page` の保存正本を参照する方針とする。

[cols="1,2,4",options="header"]
|===
^|データ ^|役割 ^|保持する主な内容

|`current.page.nodes[]`
|保存正本
|`id`, `type`, `label`, `description`, `x`, `y`, `visible`, `shape`, `size`, `style`, `resource`, `time`, `pageNumber`, `anchorHref`, `audit`

|`graph.nodes[]`
|SVG 描画用 runtime view model
|`id`, `type`, `x`, `y`, `visible`, `state`, `label`, `description`, `shape`, `size`, `style`, runtime mirror fields, `iconKind`, `thumbnailUrl`, `linkCount`

|`graph.links[]`
|SVG 描画用 link view model
|`id`, `source`, `target`, `type`, `role`, `style`, `state`

|SVG DOM
|実際に描画される SVG 要素
|`g.node`, `g.link`, `rect.shape-node`, `text.node-label`, `foreignObject`, `path`, `image` など
|===

=== graph と SVG DOM の分離

`graph.node` 及び `graph.link` は、SVG 描画に使う runtime view model である。

一方、`g.node` 及び `g.link` は、D3 により SVG 上に生成又は更新される DOM 要素である。

本表では、`graph` 欄に `graph.node` / `graph.link` の対応属性を記載し、`svg` 欄に実際の SVG DOM 要素又は D3 selection の対応を記載する。

[cols="1,2,3",options="header"]
|===
^|区分 ^|例 ^|意味

|保存正本
|`current.page.nodes[]`, `current.page.links[]`
|保存対象となる正本データである。

|graph runtime
|`graph.node`, `graph.link`
|SVG 描画、選択、ドラッグ、簡易表示に用いる軽量 runtime view model である。

|SVG DOM / D3 selection
|`g.node`, `g.link`, `rect.shape-node`, `text.node-label`, `foreignObject`
|ブラウザ上で実際に描画される SVG 要素である。
|===

=== graph に原則として持たせない項目

SVG 描画性能を確保するため、次の項目は将来的には `graph.node` に持たせない。

[cols="1,3",options="header"]
|===
^|項目 ^|理由

|`resource.original`
|原本又は参照元情報であり、SVG 描画には不要である。

|`resource.storage.files[]`
|保存ファイル群であり、重く、描画には不要である。

|`resource.viewer`
|表示操作時に解決すればよい。

|`resource.media`
|ページ数、再生時間、画像サイズなどは表示ペイン又は管理者用ペインで参照する。

|`resource.rights.*`
|権利・ライセンス・帰属情報であり、描画には不要である。

|`audit`
|管理者用ペインで参照する診断情報であり、描画には不要である。
|===

=== label と description.body

`label` は Node の短い表示名である。

`description.body` は保存正本の本文又は説明文である。

`description.format` は、`description.body` の記法を示す。値は、例えば `plain`, `asciidoc`, `markdown` などである。

現在の SVG 描画では、Memo 及び Content の `description.body` は `description.format` に従って HTML 化され、SVG の `foreignObject` 内に描画される。

[cols="1,1,2,2,4",options="header"]
|===
^|対象 ^|表示元 ^|graph ^|svg ^|内容

|Topic
|`label`
|`graph.node.label`
|`g.node text.node-label`
|概念名を表示する。`description.body` は説明であり、通常は図形内本文としては使用しない。

|Memo
|`description.body`
|`graph.node.description.body`
|`g.node foreignObject`
|Memo は `label` を持たない。`description.body` を HTML 化して図形内に表示する。

|Content
|`label` 及び `description.body`
|`graph.node.label`, `graph.node.description.body`
|`g.node text.node-label`, `g.node foreignObject`
|Content の見出しは `label`、図形内説明文は `description.body` である。`description.body` は HTML 化して `foreignObject` 内に描画する。

|Segment
|`label`
|`graph.node.label`
|`g.node text.node-label`
|Timeline Segment の表示名を表示する。

|PageMarker
|`label`
|`graph.node.label`
|`g.node text.node-label`
|Contents PageMarker の表示名を表示する。
|===

NOTE: `graph.node.text` を `description.body` の正本として扱わない。SVG 表示に必要な場合でも、保存正本は `current.page.nodes[].description.body` である。

=== resource.title の扱い

`resource.title` は定義しない。

Content の表示名は `node.label` に一本化する。

upload Content の初期 `node.label` は、登録済み original file の `file_name` 又は登録時に利用者が指定した名称から設定してよい。

HTML title、動画タイトル、PDF メタデータ title などを将来保持する場合は、`resource.metadata.extractedTitle` のような取得メタデータとして定義する。ただし、これは `node.label` とは同期させない。

=== resource.original

`resource.original` は、Content の原本又は参照元を表す。

`resource.uri` 及び `resource.canonicalUri` は新規データでは定義しない。

`remote.url` 及び `origin` は、`resource.original` に統合する。

==== upload Content の original

upload Content の場合、`resource.original` は登録済み原本ファイルを示す。

実際のファイル情報は `resource.storage.files[]` に保存する。

[source,javascript]
----
resource: {
  source: "upload",
  kind: "document",
  documentKind: "pdf",

  original: {
    type: "upload",
    storageRole: "original"
  },

  storage: {
    files: [
      {
        role: "original",
        path: "2026/05/24/file_uuid/report.pdf",
        dir_name: "data/user_uuid/upload/2026/05/24/file_uuid/",
        file_name: "report.pdf",
        mimeType: "application/pdf",
        size: 123456,
        sha256: "..."
      }
    ]
  }
}
----

==== remote Content の original

remote Content の場合、`resource.original` は外部資料の参照情報を示す。

登録時に利用した URL は `resource.original.url` に保存する。

外部資料の正規 URL は `resource.original.canonicalUrl` に保存する。

これらは Content 定義時の入力項目であり、remote Content では編集可能とする。

[source,javascript]
----
resource: {
  source: "remote",
  kind: "html",

  original: {
    type: "remote",
    url: "https://example.com/article?id=123&utm_source=mail",
    canonicalUrl: "https://example.com/article?id=123",
    accessedAt: "2026-05-24T10:00:00Z",
    identifiers: [
      { scheme: "doi", value: "10.xxxx/yyyy" },
      { scheme: "ndl", value: "..." }
    ]
  }
}
----


=== resource.viewpoint と legacy resource.contents

新規データでは `resource.contents` は使用しない。
Document / HTML Content のページ番号、anchor、viewer page 変換、PageMarker / Viewpoint に関係する metadata は `resource.viewpoint` に保存する。

旧 v2 以前のデータに `resource.contents` が存在する場合は、読み込み時又は v2 正規化時に `resource.viewpoint` へ移行し、保存時には `resource.contents` を出力しない。

== 保存時の検査

remote Content の `resource.original.url` 及び `resource.original.canonicalUrl` は、定義時入力項目であるため更新可とする。

ただし、URL 変更により viewer 解決、thumbnail 取得、PageMarker anchor、Segment、外部埋め込み可否が変わる場合がある。

保存時には次の再検査を行う。

[cols="1,3",options="header"]
|===
^|検査項目 ^|内容

|viewer 再解決
|iframe、external、PDF.js、Office Viewer、video player 等の表示方法を再判定する。

|thumbnail 再取得
|必要に応じて remote thumbnail を再取得又は再生成する。

|anchor 再検査
|HTML の場合、PageMarker の `anchorHref` と新しい URL の対応を確認する。

|Segment 再検査
|video / audio の場合、duration が変化したときに Segment の範囲を確認する。

|警告表示
|URL 変更により既存 PageMarker、Segment、thumbnail が不整合になる可能性を表示する。
|===

upload Content の原本ファイルは、`resource.storage.files[role="original"]` により管理する。

upload Content では、原本ファイルの論理パスを編集ペインで変更しない。

原本を変更する必要がある場合は、URL や path の直接編集ではなく、Content の再登録又は置換操作として扱う。
