== 目的

本仕様は、WuWei の編集ペインにおいて、選択対象の種別に応じて編集項目を整理し、必要な場合のみタブ形式で表示するための仕様を定義する。

編集対象には、Topic、Content、Memo、Link、Group、代表ノード、Timeline Segment、Contents PageMarker などがある。

これらは持つ属性の性質が異なるため、すべての対象に同じタブ構成を適用しない。

== 基本方針

編集ペインは、選択対象の種別に応じて次のいずれかの形式で表示する。

[cols="1,3", options="header"]
|===
|表示形式
|適用方針

|単一ペイン
|編集項目が表示属性中心であり、内容属性や権利属性を別タブに分ける必要がない対象に適用する。

|タブ付きペイン
|表示属性とは別に、内容属性、参照属性、典拠・権利属性などを持つ対象に適用する。
|===

基本原則は次のとおりである。

[source,text]
----
Topic、Memo、通常グループはタブを使わない。
Content、代表ノード、Segment、PageMarker は必要に応じてタブを使う。
典拠・権利タブは Content のみに適用する。
----

NOTE: Node の色、線、フォントの編集項目は、保存正本である `style.*` を更新する。`node.color`、`node.outline`、`node.outlineWidth`、`node.font` は `wuwei.draw` 用に `style` から実行時展開される runtime mirror field であり、編集ペインの保存項目として扱わない。編集ペインで `style.*` を変更した場合は、描画更新前に runtime mirror field を再展開する。

NOTE: 編集ペインで編集する対象は、保存対象となる正本データである。表示スタイルのうち、塗り色、線色、線幅、線種、フォント、ラベル行数、ラベル位置は `style.*` を編集する。`shape` 及び `size` は当面 Node 直下の保存項目として扱い、編集ペインでも直下の `shape` 及び `size` を更新する。`x`、`y`、`visible` は保存対象であるが、編集ペインでは変更しない。`x` 及び `y` はノートキャンバス上のドラッグ操作で変更し、`visible` はコンテキストメニュー右上の `wilt` / `bloom` / `root` / `hide` により変更する。`linkCount` は Node 直下の表示補助データとして扱うが、正本は `links[]` であり、必要に応じて再計算してよい。

== タブ分類

タブを使用する場合、基本タブは次の三つとする。

[cols="1,3", options="header"]
|===
|タブ
|意味

|表示
|画面上の見た目、サイズ、色、線、フォントなどを編集する。配置座標 `x`, `y` は編集ペインでは編集しない。

|内容
|対象が表す意味内容、参照先、ページ番号、時間範囲などを編集する。

|典拠・権利
|Content の典拠情報及び `resource.rights` に定義された権利情報を編集する。
|===

ただし、対象種別によって表示するタブは異なる。


== 配置座標の編集方針

編集ペインでは、ノードの配置座標 `x`, `y` を編集項目として表示しない。

ノードの位置変更は、ノートキャンバス上で対象ノードをドラッグして行う。

[source,text]
----
位置変更:
  編集ペインでは行わない
  ノートキャンバス上のドラッグ操作で行う
----

必要に応じて、開発者向け又は診断表示として `x`, `y` を read only 表示することはできる。ただし、通常の利用者向け編集項目には含めない。

Segment 及び PageMarker の `axisPos` も、編集ペインでは直接入力しない。Timeline axis 又は Contents axis 上でのドラッグ操作又は専用の軸操作により変更する。

== 対象種別ごとの編集ペイン仕様

=== Topic

Topic は、通常の概念ノードである。

Topic ではタブ制御を行わない。

Topic の label や description は、形状、色、フォントなどと同じ編集ペイン内で扱う。

[cols="1,3", options="header"]
|===
|項目群
|編集項目

|内容表示
|`label`, `description`, `description.format`

|形状
|`shape`, `size.width`, `size.height`, `size.radius`

|ラベル表示
|`text.position`, `text.width`, `text.height`, `style.font`, `style.label`

|色・線
|`style.fill`, `style.line.color`, `style.line.width`, `style.line.kind`
|===

Topic では、`label` は概念名であり、`description.body` はその説明である。

=== Memo

Memo は、付箋・メモ用ノードである。

Memo ではタブ制御を行わない。

Memo は `label` を持たない。Memo の表示本文は `description.body` として定義する。

旧データにおいても Memo には `label` は定義されていない。したがって、Memo に `label` が存在する場合は、共通 Node 正規化処理等により付与された不要項目とみなし、読み込み時又は保存時に無視する。

[cols="1,3", options="header"]
|===
|項目群
|編集項目

|本文
|`description.body`, `description.format`

|形状
|`shape`, `size.width`, `size.height`

|Memo 表示
|`style.memo.corner`, `style.memo.foldSize`

|色・フォント
|`style.fill`, `style.font.family`, `style.font.size`, `style.font.color`

|線
|`style.line.color`, `style.line.width`, `style.line.kind`
|===

Memo の検索見出しや一覧表示が必要な場合は、`description.body` の先頭文字列を用いる。

=== Content

Content は、PDF、Web、画像、動画、Office 文書などを表すノードである。

Content は、表示属性に加えて `resource`、`viewer`、`storage`、`contents`、`rights` など多くの内容属性を持つため、タブ制御を行う。

Content のタブ構成は次のとおりである。

[source,text]
----
表示
内容
典拠・権利
----

==== Content / 表示タブ

[cols="1,3", options="header"]
|===
|項目群
|編集項目

|基本表示
|`label`, `description.body`, `description.format`

|形状
|`shape`, `size.width`, `size.height`, `size.radius`

|サムネイル表示
|`style.thumbnail`。`resource.thumbnailUri` はサーバー登録又は生成情報として read only

|ラベル表示
|`text.position`, `text.width`, `style.font`, `style.label`

|色・線
|`style.fill`, `style.line.color`, `style.line.width`, `style.line.kind`
|===

==== Content / 内容タブ

[cols="1,1,3", options="header"]
|===
|項目群
|編集可否
|編集項目

|Resource 基本
|一部 read only
|`resource.source`, `resource.kind`, `resource.documentKind`, `resource.videoKind`

|URI
|read only
|`resource.uri`, `resource.canonicalUri`

|タイトル
|read only
|`resource.title`

|MIME
|read only
|`resource.mimeType`

|Media
|read only
|`resource.media`

|Viewer
|read only
|`resource.viewer.defaultMode`, `resource.viewer.supportedModes`, `resource.viewer.embed`

|Storage
|read only
|`resource.storage.files[]`

|Contents 設定
|一部可
|`resource.contents.firstPageNumber` のみ編集可。`resource.contents.pageCount` は `resource.media.pageCount` と同じ物理ページ数として read only。`resource.contents.pageOffset` は `firstPageNumber - 1` で算出される派生値として read only。`pageMin` 及び `pageMax` フィールドは定義しない
|===

`mimeType`, `storage.files[]`, `viewer`, `media` など、サーバー登録時又はプログラム判定時に確定する情報は、Content node の `resource` として保存するが、編集ペインでは変更させない。これらの詳細は情報ペインで表示する。

NOTE: PDF / Office 等の文書 Content では、編集ペインで利用者が変更できるページ関連項目は `resource.contents.firstPageNumber` のみとする。`resource.contents.pageOffset` は `firstPageNumber - 1` として算出し、`resource.contents.pageCount` は `resource.media.pageCount` と同じ物理ページ数として read only 表示する。最小ページ番号及び最大ページ番号は独立した保存項目として定義しないため、`pageMin` / `pageMax` フィールド又は入力欄は設けない。PageMarker の入力可能範囲は、必要に応じて `firstPageNumber` と `pageCount` から一時的に算出する。

`resource.title`, `resource.uri`, `resource.canonicalUri`, `resource.source`, `resource.kind`, `resource.documentKind`, `resource.videoKind` は v2 Resource の識別・参照・分類情報である。通常はサーバー登録時又はプログラム判定時に確定するため、編集ペインでは read only とする。手動補正を将来認める場合でも、明示的な補正 UI として扱い、fallback 判定には使わない。

==== Content / 典拠・権利タブ

典拠・権利タブは Content のみに表示する。

このタブは、v2 Resource 仕様に合わせて、典拠情報として `resource.title`, `resource.uri`, `resource.canonicalUri` を read only で表示し、権利情報として編集可能な `resource.rights` を扱う。`resource.credit` 及び `resource.licence` は現在の v2 データ仕様では使用しない。

[cols="1,1,1,3", options="header"]
|===
|区分
|項目
|入力形式
|対応するデータ項目

|典拠
|タイトル
|read only
|`resource.title`

|典拠
|参照 URI
|read only
|`resource.uri`

|典拠
|正規 URI
|read only
|`resource.canonicalUri`

|権利
|権利者
|input text
|`resource.rights.owner`

|権利
|著作権表示
|textarea
|`resource.rights.copyright`

|権利
|ライセンス
|input text / select
|`resource.rights.license`

|権利
|帰属表示
|textarea
|`resource.rights.attribution`
|===

`resource.title`, `resource.uri`, `resource.canonicalUri` は、サーバー登録又は参照元取得時に確定する識別・参照先情報であるため、編集ペインでは read only とする。

`resource.rights` は単一 textbox ではなく、項目ごとの input、select、textarea として編集する。内訳は次のとおりである。

[source,javascript]
----
resource: {
  rights: {
    owner: "",        // 権利者、所有者、著作者、提供元など
    copyright: "",    // 著作権表示、Copyright notice
    license: "",      // ライセンス名又は利用条件
    attribution: ""   // 表示すべき帰属・クレジット文
  }
}
----


=== Link

Link は、ノード間の関係を表す。

Link はタブ制御を行わない。

Link の編集ペインでは、関係構造と表示 style を同一ペイン内に表示する。ただし、共同編集モード又は imported ノートでは、他者作成 Link の関係構造は read only とする。

[cols="1,3", options="header"]
|===
|項目群
|編集項目

|関係
|`source`, `target`, `type`, `role`

|表示
|`style.lineColor`, `style.lineType`, `style.width`, `style.thickness`, `style.preset`

|状態
|`state`
|===

`style.preset` には、`s`, `v`, `h`, `v2`, `h2` などを含める。

=== 通常グループ

通常グループは、Timeline 及び Contents 以外の `s`, `v`, `h`, `v2`, `h2` などのグループである。

通常グループではタブ制御を行わない。

label、description、グループ表示、軸線、外周線、代表ノード style を同一ペイン内で扱う。

[cols="1,3", options="header"]
|===
|項目群
|編集項目

|基本
|`label`, `description`

|グループ構造
|`group.type`, `members`

|軸線
|`style.axis.lineColor`, `style.axis.lineType`, `style.axis.width`

|外周線
|`style.perimeter.lineColor`, `style.perimeter.lineType`, `style.perimeter.width`

|代表ノード表示
|representative node `style`
|===

imported ノート又は team ノートで他者定義グループを扱う場合、グループ構造は変更不可とする。変更したい場合は、既存グループを削除扱いにして履歴管理し、新規グループとして再定義する。

ただし、許可される場合は、軸線、外周線、代表ノードの style は変更可能とする。

=== グループ代表ノード

グループ代表ノードは、`type: "Topic"` の特殊用途ノードとして扱う。

代表ノードであることは、`type` ではなく、`topicKind`, `groupRole`, `representativeOf`, `groupRef` により示す。

[source,javascript]
----
{
  type: "Topic",
  topicKind: "group-representative",
  groupRole: "representative",
  groupRef: "_group_uuid",
  representativeOf: {
    kind: "group",
    id: "_group_uuid"
  }
}
----

代表ノードは、通常の Topic と異なり、グループ構造を代表するノードである。

代表ノードについては、必要に応じてタブ制御を行う。ただし、基本は表示中心である。

[source,text]
----
表示
内容
----

== collabNoteState による編集制御

編集ペインは、ノート状態 `collabNoteState` に応じて入力可否を切り替える。

[cols="1,3", options="header"]
|===
|collabNoteState
|編集方針

|`own`
|自分で作ったノート。通常編集を許可する。

|`imported`
|他者から入手して import したノート。元の作成者が作った意味属性は保護し、許可された表示調整及び補足を中心に編集する。

|`team`
|チーム共同編集ノート。他者作成要素の変更及び削除は不可。必要な指摘は Memo 又はチャットで伝える。
|===

== タブ表示判定

編集対象ごとのタブ表示判定は次のとおりとする。

[cols="1,2,2", options="header"]
|===
|対象
|タブ
|備考

|Topic
|なし
|表示ペインのみ。label と description も同一ペイン内で扱う。

|Memo
|なし
|label なし。description.body を本文として扱う。

|Content
|表示 / 内容 / 典拠・権利
|典拠・権利は Content のみ。

|Link
|なし
|関係と style を同一ペイン内で扱う。

|通常グループ
|なし
|構造と表示 style を同一ペイン内で扱う。ただし他者定義構造は read only。

|代表ノード
|表示 / 内容
|内容タブは主に read only。imported/team では style 中心。

|Segment
|表示 / 内容
|時間情報を内容タブに分離。

|PageMarker
|表示 / 内容
|ページ番号、anchor、文書参照を内容タブに分離。
|===

== 実装方針

編集ペイン生成処理では、対象種別に応じて次の判定を行う。

[source,javascript]
----
function getEditPaneSpec(target) {
  if (target.type === 'Content') {
    return {
      tabs: ['display', 'content', 'rights']
    };
  }

  if (target.type === 'Segment') {
    return {
      tabs: ['display', 'content']
    };
  }

  if (target.type === 'PageMarker') {
    return {
      tabs: ['display', 'content']
    };
  }

  if (isRepresentativeNode(target)) {
    return {
      tabs: ['display', 'content']
    };
  }

  return {
    tabs: null
  };
}
----

`tabs: null` の場合は、従来どおり単一編集ペインを表示する。

== まとめ

編集ペインのタブ対応では、すべての対象に一律タブを適用しない。

Topic、Memo、Link、通常グループは、編集項目が比較的単純であるため、単一ペインとする。

Content、代表ノード、Segment、PageMarker は、表示属性とは別に内容属性を持つため、タブ付きペインとする。

典拠・権利タブは Content のみに適用する。

Memo は `label` を持たず、`description.body` を表示本文として扱う。
