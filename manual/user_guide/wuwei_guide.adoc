このガイドは、2026-05-01 時点の WuWei の利用方法とデータの扱いを説明します。

== WuWei の考え方

WuWei は、資料を単に保存するためのツールではありません。

PDF、Office 文書、画像、Web ページなどをコンテンツとして登録し、それをどのように読み、どの観点で重要だと考えたかをノート、トピック、リンクとして残すための作業環境です。

ノートは完成した結論だけでなく、調査途中の仮説、気づき、比較、関係づけも記録します。

== 基本概念

=== ノート

ノートは WuWei の作業単位です。

ノートには次の情報を保存します。

* ノート名
* 説明
* ページ
* グループ
* ノード
* リンク
* 参照するコンテンツ
* ノートサムネイル

ノート保存では、アップロード済みファイル本体をノートディレクトリへコピーしません。ノート JSON には参照を保存します。

=== ページ

ページはノート内の作業面です。

大きな調査テーマを複数のページに分けて整理できます。

=== ノード

ノードはページ上に配置される要素です。

主なノード種別は次の通りです。

* アップロードコンテンツ
* トピック
* 付箋
* グループ
* リンクの接続点

アップロードコンテンツのノードは、ファイル本体を直接保持せず、ノート内の `resources[]` を参照します。

=== リソース

リソースは、ノードが参照する再利用可能なコンテンツ定義です。

アップロードされたファイルは次の場所に保存されます。

[source,text]
----
data/{user_uuid}/upload/YYYY/MM/DD/{upload_uuid}/
  manifest.json
  {original_file}
  thumbnail.jpg
  preview.pdf
----

`manifest.json` がアップロードコンテンツの正本メタデータです。

== ホーム画面

ホーム画面では、登録済みコンテンツを探して選択できます。

検索対象は `upload/YYYY/MM/DD/{upload_uuid}/manifest.json` を持つコンテンツです。

日付範囲とキーワードで絞り込みできます。

* 年
* 月
* 日
* 開始日
* 終了日
* キーワード

ホーム画面から選択したコンテンツは、現在編集中のノートへ追加できます。また、選択したコンテンツだけを含む新規ノートを作成できます。

== アップロード

ファイルをアップロードすると、WuWei は次を作成します。

* 元ファイル
* `manifest.json`
* `thumbnail.jpg`
* Office 文書の場合は `preview.pdf`

同じ SHA-256 の元ファイルは同じコンテンツとみなし、重複保存を避けます。

Office 文書は LibreOffice で PDF preview を生成し、その先頭ページからサムネイルを作成します。

== ノート保存

ノート保存では、ノート JSON を次に保存します。

[source,text]
----
data/{user_uuid}/note/YYYY/MM/DD/{note_uuid}/note.txt
----

アップロード済みコンテンツは `upload/` に置いたまま、ノート JSON には参照を保存します。

これにより、保存処理を軽くし、大きなファイルの重複コピーを避けます。

== ノート一覧

ノート一覧には、`note.txt` が存在し、読み込み可能なノートだけが表示されます。

保存途中で失敗した空ディレクトリや、`ERROR NOTE NOT FOUND` になるディレクトリは表示しません。

未保存作業の救済が必要な場合のみ、明示的なオプションで `new_note` を対象にします。

== ノートのダウンロード

ノートのダウンロードでは、保存済みノート JSON が参照しているアップロードコンテンツを収集して ZIP 化します。

ZIP には次が含まれます。

* `note.txt`
* ノートサムネイル
* 参照先の `upload/YYYY/MM/DD/{upload_uuid}/` ディレクトリ

通常保存時にはファイルをコピーせず、ダウンロード時に必要な参照先を集めます。

== ノートのアップロード

ダウンロードした ZIP をアップロードすると、WuWei はノート JSON と参照先 upload resource を復元します。

既に同じ SHA-256 のコンテンツが存在する場合は、既存の upload resource を再利用できます。

== ファイル表示

ブラウザから `data/` 配下へ直接アクセスしません。

ファイル表示は `load-file.cgi` を通して行います。

[source,text]
----
load-file.cgi?area=upload&path=YYYY/MM/DD/{upload_uuid}/thumbnail.jpg&user_id={user_uuid}
----

`area=upload` の相対パスを使うことで、サムネイル、preview PDF、元ファイルを安全に参照します。

== 旧形式データ

開発途中で作成された旧形式データは、今後の通常一覧には表示しません。

表示しない例は次の通りです。

* `manifest.json` がない upload ディレクトリ
* `resource.json` のみの旧 resource
* `note/new_note/resource/` にだけ存在するサムネイル
* `note.txt` が存在しないノートディレクトリ

今後は manifest backed resource を正本とし、旧形式救済のために保存処理を複雑化しません。

== 運用上の注意

EC2 では `data/` と `server/log/` を nginx 実行ユーザーが書き込めるようにします。

[source,bash]
----
sudo chown -R nginx:nginx data server/log
chmod +x server/*.cgi server/is-login server/bin/*
----

ユーザー認証情報は `public_html` 配下ではなく、ブラウザから直接アクセスできない場所に置きます。
