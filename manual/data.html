WuWei データ定義・保存設計

1. 目的

この文書は、WuWei app2 で扱う Note、Page、Node、Link、Group、Resource のデータモデルと、保存時のファイル配置を一体として定義する。

旧バージョンの Note データは wuwei.note が読み込む時点で新モデルへ変換する。 変換後の実行時データおよび保存データには、旧モデル由来の属性を残さない。 旧モデル属性が残っている場合は互換処理を行わず、エラーとして扱う。

2. 基本方針

  • Note は resources[] と pages[] を持つ。

  • Page は nodes、links、groups を持つ。

  • Content ノードは Resource 本体を直接保持せず、resourceRef で resources[].id を参照する。

  • 同じ Resource は複数の Content ノードから参照できる。

  • 同一 Resource の参照ページ、シート、時刻範囲、fragment などの違いは Content 側の resourceView に保持する。

  • 外部 Web Resource は原則として実体コピーせず、メタデータ、URI、埋め込み情報を保持する。

  • 自己アップロード Resource は Resource ライブラリに登録し、必要に応じて Note snapshot にコピーする。

  • Page ID は _page_uuid とし、ページ番号は pages[] の表示順から計算する。

  • Link の矢印は routing.startArrow と routing.endArrow で表す。

  • Group 種別は simple、horizontal、vertical、timeline とする。

  • Group の表示/非表示は、Group 全体を一括で制御する。

3. 識別子

ID は UUID 文字列を基本とし、WuWei 内では先頭に _ を付ける。

_xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

主な ID は次のとおりである。

ID 用途

note_id

Note を識別する。

page.id

Page を識別する恒久 ID。ページ番号ではない。

node.id

Node を識別する。

link.id

Link を識別する。

group.id

Group を識別する。

resource.id

Resource を識別する。

4. 共通型

4.1. Audit

{
  "owner": "sambuichi",
  "createdBy": "_user_uuid",
  "createdAt": "2026-04-27T12:00:00+09:00",
  "lastModifiedBy": "",
  "lastModifiedAt": ""
}

4.2. RichText

説明、注釈、Memo 本文などの文字列は、形式と本文を分けて保持する。

{
  "format": "asciidoc",
  "body": "== TITLE"
}

format は plain/text、asciidoc、markdown、html などを指定できる。 表示時は format に従ってレンダリングする。 検索時は markup 制御文字を除いた本文相当の文字列を対象にする。

4.3. Transform

Page または描画キャンバスの表示位置と倍率を表す。

{
  "x": 0,
  "y": 0,
  "scale": 1
}

4.4. Style

{
  "fill": "#FEFEFE",
  "font": {
    "family": "sans-serif",
    "size": 14,
    "color": "#000000",
    "align": "center"
  },
  "line": {
    "kind": "SOLID",
    "color": "#666666",
    "width": 1
  }
}

編集画面の HTML id や変数名では、階層名を _ で連結した形式を用いる。 例: style.fill は style_fill、style.font.size は style_font_size。

5. Note

Note はページ、参照 Resource 定義、監査情報を持つ。 Note 単体をファイルとして受け渡しても再現できるように、参照 Resource の定義コピーを resources[] に保持する。

{
  "note_id": "_note_uuid",
  "note_name": "Note label",
  "description": "Note description",
  "currentPage": "_page_uuid",
  "resources": [],
  "pages": [],
  "audit": {}
}
項目 説明

note_id

Note ID。

note_name

Note 名。

description

Note 全体の説明。

currentPage

現在表示している Page ID。ページ番号ではなく _page_uuid を保存する。

resources

この Note が参照する Resource 定義のコピー。

pages

表示順に並ぶ Page 配列。

audit

監査情報。

6. Page

Page は、1 枚の作業面を表す。 Page は Node、Link、Group、表示位置、thumbnail、監査情報を持つ。

Page にはページ番号を恒久 ID として持たせない。 Page の恒久 ID は id に保持する _page_uuid である。 ページ番号は Note の pages[] 内の順序から 1 始まりで計算される表示用番号である。 ページの追加、削除、並べ替えにより、同じ Page ID でも表示用ページ番号は変わり得る。

{
  "id": "_page_uuid",
  "label": "Page label",
  "description": {
    "format": "plain/text",
    "body": ""
  },
  "nodes": [],
  "links": [],
  "groups": [],
  "transform": {
    "x": 0,
    "y": 0,
    "scale": 1
  },
  "thumbnail": null,
  "audit": {}
}
項目 説明

id

Page の恒久 ID。

label

Page 名。画面表示や Page 一覧で使用する。

description

Page の説明。

nodes

Page 上に配置される Node の配列。

links

Page 上の Node 間を結ぶ Link の配列。

groups

Page 上の Group 定義の配列。

transform

Page の表示位置と倍率。

thumbnail

Page 一覧用の縮小表示。

audit

監査情報。

6.1. Page 番号

ページ番号は保存値ではなく、pages[] の現在の順序から求める。

pageNumber = pages.indexOf(page) + 1

Page 一覧、draw 上の表示、ページ操作 UI ではこの表示番号を使う。 保存データの参照や currentPage には Page ID を使う。

6.2. Page 追加

Page メニューの「新規ページ」は、新しい Page ID を採番し、pages[] の末尾に空の Page を追加する。 追加直後は、新しく作成された Page を currentPage に設定する。

{
  "id": "_new_page_uuid",
  "label": "",
  "description": {
    "format": "plain/text",
    "body": ""
  },
  "nodes": [],
  "links": [],
  "groups": [],
  "transform": {
    "x": 0,
    "y": 0,
    "scale": 1
  },
  "thumbnail": null,
  "audit": {}
}

ページ追加時に既存 Page の番号を ID として再利用してはならない。 削除された Page の表示番号が空いていても、Page ID は常に新しい UUID とする。

6.3. Page 削除

Page 削除では、対象 Page を pages[] から取り除く。 削除後のページ番号は残った pages[] の順序から再計算する。 削除対象が currentPage の場合は、残っている先頭 Page を新しい currentPage とする。 すべての Page が削除される場合は、空の Page を 1 件作成する。

6.4. Page 複製

Page 複製では、Page ID、Node ID、Link ID、Group ID を新しく採番する。 複製元 Page の Node/Link/Group をそのまま ID 付きで共有してはならない。 Content が参照する Resource は同じ Resource ID を共有してよい。

7. Resource

Resource は Content ノードから参照される再利用可能なコンテンツ定義である。 Resource は大きく、外部 Web Resource と Uploaded Resource に分ける。

分類 説明

外部 Web Resource

Web page、blog、YouTube、Vimeo、Entity site、公開 PDF URL など、他者の公開情報を参照する。

Uploaded Resource

PDF、Office、text、video、audio、photo など、ユーザーがアップロードした自己登録情報。

7.1. Resource 定義

{
  "id": "_resource_uuid",
  "type": "Resource",
  "origin": {
    "type": "userRegistered",
    "subtype": "",
    "provider": ""
  },
  "identity": {
    "title": "Document title",
    "uri": "",
    "canonicalUri": ""
  },
  "media": {
    "kind": "document",
    "mimeType": "application/pdf",
    "downloadable": true,
    "duration": null
  },
  "viewer": {
    "supportedModes": ["infoPane", "newTab", "newWindow", "download"],
    "defaultMode": "infoPane",
    "embed": {
      "uri": "",
      "html": ""
    }
  },
  "storage": {
    "managed": true,
    "copyPolicy": "snapshot",
    "sourcePath": "yyyy/mm/dd/original.pdf",
    "primaryPath": "data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}",
    "snapshotPath": "",
    "files": []
  },
  "snapshotSources": {},
  "rights": {
    "owner": "",
    "copyright": "",
    "license": "",
    "attribution": ""
  },
  "audit": {}
}

7.2. media.kind

値 説明

webpage

通常の Web ページ。

video

YouTube、Vimeo、アップロード動画。

audio

音声。

document

PDF、Office、text などの文書。

image

画像。

general

特定の viewer を持たない一般 Resource。

YouTube と Vimeo は webpage ではなく video として扱う。

7.3. Resource URI の解決順序

info ペイン、外部ウィンドウ表示、Resource 追加処理で表示 URI を求めるときは、次の順序で解決する。

resource.identity.uri
resource.viewer.embed.uri
resource.snapshotSources.previewUri
resource.identity.canonicalUri
resource.snapshotSources.originalUri

identity.uri は通常表示 URL である。 YouTube や Vimeo など、通常 URL と iframe 埋め込み URL が異なる場合は、通常 URL を identity.uri、iframe 用 URL を viewer.embed.uri に保持する。

URI が完全に空の Resource は、info ペインに表示対象を持たない。 表示側は URL を推測してはならない。

7.4. iframe 表示

info ペインで Web page、YouTube、Vimeo などを表示する場合、PHP proxy は使用しない。 server/proxy.php などの PHP スクリプトを iframe の src に設定してはならない。 サーバー CGI は shell script を基本とし、PHP 実行環境を前提にしない。

Web page は URI を直接 iframe に設定する。 対象サイトが X-Frame-Options や CSP により iframe 表示を拒否する場合は、iframe を隠し、URL と拒否メッセージを表示する。 この場合でも、新しいタブまたは別ウィンドウで開くリンクを提供する。

7.5. storage

storage は Resource の保存場所とコピー方針を定義する。

項目 説明

managed

WuWei がローカルファイルを管理する場合は true。

copyPolicy

snapshot または metadataOnly。

sourcePath

upload 領域からの相対パス。例: 2026/04/26/document.pdf。

primaryPath

Resource ライブラリ側の保存ディレクトリ。

snapshotPath

Note snapshot 側の Resource 保存ディレクトリ。

files

Resource に関連するファイル一覧。

storage.files[] の例:

[
  {
    "role": "original",
    "path": "yyyy/mm/dd/original.pdf",
    "sourcePath": "C:/Apache24/htdocs/wu_wei2/data/{user_uuid}/upload/yyyy/mm/dd/original.pdf",
    "mimeType": "application/pdf",
    "size": 123456,
    "sha256": "..."
  },
  {
    "role": "thumbnail",
    "path": "thumbnail.png",
    "mimeType": "image/png",
    "size": 12345,
    "sha256": "..."
  },
  {
    "role": "preview",
    "path": "preview.pdf",
    "mimeType": "application/pdf",
    "size": 123456,
    "sha256": "..."
  }
]

7.6. Resource の同一判定

Resource ライブラリでは、同じ実体を重複保存しない。 同一判定は次の優先順位で行う。

resource.identity.canonicalUri
resource.identity.uri
resource.storage.sourcePath

upload ファイルでは、yyyy/mm/dd/filename が同じであれば、sha256 が異なっても同じファイルの更新とみなす。 Note snapshot 配下では、同じ Resource であっても Note ごとにコピーを持ってよい。 Note 単体を受け渡しても再現可能にするためである。

7.7. 空 URI による上書き禁止

既存 Resource に URI が定義されている場合、Note 保存または Resource 更新で渡された Resource 定義の URI が空であっても、既存 URI を空で上書きしてはならない。

保護対象は次の項目である。

resource.identity.uri
resource.identity.canonicalUri
resource.viewer.embed.uri
resource.snapshotSources.previewUri
resource.snapshotSources.originalUri
resource.snapshotSources.thumbnailUri

8. Content と ResourceView

Content は Resource を参照する Node である。 Resource の実体や定義を Content ノードに展開して永続保存しない。

{
  "id": "_content_uuid",
  "type": "Content",
  "resourceRef": "_resource_uuid",
  "resourceView": {
    "mode": "documentPage",
    "page": 12,
    "fragment": "#page=12",
    "time": {
      "start": "",
      "end": ""
    }
  },
  "x": 0,
  "y": 0,
  "shape": "RECTANGLE",
  "size": {
    "width": 120,
    "height": 100
  },
  "visible": true,
  "label": "Content label",
  "description": {
    "format": "asciidoc",
    "body": ""
  },
  "style": {},
  "audit": {}
}

label と description は Note 上の解釈や注釈であり、Resource の identity.title とは独立している。

9. Node

9.1. 共通 Node

{
  "id": "_node_uuid",
  "type": "Topic",
  "x": 0,
  "y": 0,
  "shape": "RECTANGLE",
  "size": {
    "width": 120,
    "height": 40
  },
  "visible": true,
  "label": "",
  "description": {
    "format": "asciidoc",
    "body": ""
  },
  "style": {},
  "audit": {}
}

9.2. Topic

Topic は概念、タグ、話題を表す Node である。

9.3. Content

Content は Resource を参照する Node である。 resourceRef は必須である。

9.4. Memo

Memo は付箋紙として表示する Node である。 本文は description.body に保存し、description.format に従ってレンダリングする。

10. Link

Link は Node 間の接続を表す。

{
  "id": "_link_uuid",
  "type": "Link",
  "from": "_source_node_uuid",
  "to": "_target_node_uuid",
  "shape": "NORMAL",
  "x": 0,
  "y": 0,
  "x2": 0,
  "y2": 0,
  "visible": true,
  "label": "",
  "description": {
    "format": "plain/text",
    "body": ""
  },
  "style": {},
  "routing": {
    "startArrow": {
      "kind": "NONE",
      "size": 12
    },
    "endArrow": {
      "kind": "ARROW",
      "size": 12
    }
  },
  "audit": {}
}

shape は NORMAL、HORIZONTAL、HORIZONTAL2、VERTICAL、VERTICAL2 などを使用できる。 x と y は第 1 制御点である。 HORIZONTAL2 と VERTICAL2 は第 2 制御点として x2 と y2 を保存する。 NORMAL、HORIZONTAL、VERTICAL では通常 x2 と y2 は使用しない。

11. Group

Group は複数の Node、Link、または Group 軸をまとめて扱うための定義である。

{
  "id": "_group_uuid",
  "type": "simple",
  "visible": true,
  "members": [
    {
      "nodeId": "_node_uuid",
      "role": "member",
      "order": 1
    }
  ],
  "spine": {},
  "style": {},
  "audit": {}
}

Group 種別は次のいずれかとする。

  • simple

  • horizontal

  • vertical

  • timeline

Group の表示/非表示は Group 全体に対して適用する。 Group のメンバー Node のいずれかを Group 操作として非表示にする場合、その Group 全体を非表示にする。 同じ規則は timeline Group にも適用する。

11.1. simple Group

simple Group はメンバー全体を囲むグループである。 囲み線はメンバーの現在位置に追従して描画する。

11.2. horizontal / vertical Group

horizontal Group と vertical Group は軸を持つグループである。 軸は表示対象であり、ドラッグ操作により移動できる。 simulation モードでは、Group の形状を不用意に変形しない。

11.3. timeline Group

timeline Group は時間軸を持つグループである。 開始時刻、終了時刻、軸方向、表示位置、対象 media への参照を保持する。 timeline Group も Group 一括表示/非表示の対象である。

12. 保存ファイル構造

12.1. Note

data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/note.json
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/resource.json
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/original.*
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/thumbnail.*
data/{user_uuid}/note/yyyy/mm/dd/{note_uuid}/resource/{resource_uuid}/preview.pdf

note.json は metadata と Note JSON 本体を保持する。 Note JSON 本体は json_base64 として保存できる。

format_version 2
id _note_uuid
user_id _user_uuid
name Note label
description Note description
thumbnail ...
saved_at 2026-04-27T12:00:00+0900
json_encoding base64
json_base64 eyJub3RlX2lkIjoiX25vdGVfdXVpZCJ9...

json_base64 を decode した Note 本体は、この文書の Note 定義に従う。

12.2. Resource library

data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/resource.json
data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/original.*
data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/thumbnail.*
data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/preview.pdf

同じ Resource は Resource library では重複保存しない。 一方、Note snapshot 配下では Note ごとにコピーを持ってよい。

12.3. Upload

upload 済みファイルの実体は次の場所にある。

data/{user_uuid}/upload/yyyy/mm/dd/{filename}

Resource は upload 実体を直接複製するのではなく、storage.sourcePath や storage.files[].sourcePath で参照する。

13. save-note / load-note

13.1. save-note

save-note は Note 保存時に、Note 内で新規作成または更新された Resource も保存する。 既存 Resource を参照するだけの場合でも、Note 側には Resource 定義コピーを resources[] として保持する。

自己アップロードファイルは Resource library と Note snapshot の両方に必要なファイルを持てる。 外部 Web URL は原則コピーせず、metadata と embed 情報のみを保持する。

13.2. load-note

load-note は Note ファイルを解決して読み込む。 Note snapshot 配下の resource.json は Note 一覧の対象に含めない。

読込後、wuwei.note は resources[] を id で索引化し、各 Content ノードの resourceRef に対応する Resource を実行時用 node.resource として復元する。 復元された node.resource は実行時専用であり、保存時の Note JSON には残さない。

14. wuwei.home

wuwei.home は Resource library に登録された Resource の一覧を表示し、選択した Resource を Note に追加するための画面である。 対象は data/{user_uuid}/resource/yyyy/mm/dd/{resource_uuid}/resource.json とする。 upload ディレクトリやテストデータを直接一覧対象にしない。

ギャラリーと選択中コンテンツ preview は次の順序で表示 URI を解決する。

resource.viewer.embed.uri
resource.snapshotSources.previewUri
resource.identity.uri
resource.identity.canonicalUri
storage.files[].role = "preview"
storage.files[].role = "thumbnail"

Add to note は、選択中 Resource を編集中 Note に Content ノードとして追加する。 新規ノート は、新しい Note を作成してから選択中 Resource を Content ノードとして追加する。

ギャラリーのカードには Resource の thumbnail または preview を表示する。 Note の一覧やギャラリーで Note を表示する場合は、Note の currentPage に対応する Page を優先して thumbnail 表示する。

15. 検索対象

Resource 検索の初期対象は次の項目とする。

  • resource.label

  • resource.description.body

  • resource.identity.title

  • resource.storage.sourcePath

  • resource.storage.files[].path

  • resource.identity.uri

  • resource.identity.canonicalUri

media.kind、media.mimeType、rights.owner、rights.copyright、rights.license、rights.attribution は将来の検索対象候補とし、初期実装では検索対象外とする。

16. 禁止する旧モデル属性

wuwei.note の読み込み変換後、次の属性は実行時データおよび保存データに残してはならない。

  • node.name

  • node.resource

  • node.media

  • node.uri

  • node.url

  • node.format

  • node.hidden

  • node.view

  • link.source

  • link.target

  • link.hidden

  • link.view

  • note.resources がオブジェクト map である構造

  • page.pp

  • ページ番号を Page ID として扱う構造

  • associations

  • idx

  • owner_id

  • ownerId

  • userId

  • userID

  • nFont_size

  • nFont_color

  • background

対応する新モデルの保存先は次のとおりである。

旧属性 新モデル

node.name

node.label

node.media.kind

resource.media.kind

node.media.mimeType

resource.media.mimeType

node.url / node.uri

resource.identity.uri、resource.identity.canonicalUri

node.format

node.description.format

node.content

node.description.body

nFont_size

node.style.font.size

nFont_color

node.style.font.color

background

node.style.fill
