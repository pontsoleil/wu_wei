= WuWei ホーム画面仕様
:toc:
:toclevels: 3
:sectnums:

この文書は、WuWei ホーム画面の現行仕様を定義します。

== 目的

ホーム画面は、登録済みコンテンツを探し、選択し、編集中のノートへ追加するための画面です。

対象コンテンツは次の通りです。

* アップロード済みファイル
* PDF
* Office 文書
* 画像
* 今後登録される Web ページ等の外部コンテンツ

== 正本データ

ホーム画面のコンテンツギャラリーは、次の manifest を正本として読み込みます。

[source,text]
----
data/{user_uuid}/upload/YYYY/MM/DD/{upload_uuid}/manifest.json
----

`resource/YYYY/MM/DD/{resource_uuid}/resource.json` だけを持つ旧形式データは表示対象外です。

== 表示カード

各カードは `manifest.json` から次の情報を作成します。

[cols="1,2"]
|===
|表示項目 |取得元

|タイトル
|`title` または `original.display_name`

|登録日時
|`created_at`

|メディア種別
|`kind`

|サムネイル
|`thumbnail.file`

|プレビュー
|`preview.file`

|元ファイル
|`original.file`
|===

サムネイル URL は `load-file.cgi` 経由で生成します。

[source,text]
----
area=upload
path=YYYY/MM/DD/{upload_uuid}/thumbnail.jpg
----

サムネイルが存在しない場合は、壊れた画像を出さず、種別アイコンにフォールバックします。

== 検索条件

ホーム画面では、日付範囲とキーワードでコンテンツを絞り込みます。

[cols="1,2"]
|===
|条件 |意味

|`year`
|年単位の絞り込み

|`month`
|月単位の絞り込み

|`date`
|日単位の絞り込み

|`start_date`
|開始日

|`end_date`
|終了日

|`term`
|タイトル、ファイル名、説明、コメントのキーワード
|===

カレンダーは `upload/**/manifest.json` が存在する日付を登録日として扱います。

== CGI

ホーム画面は次の CGI を使います。

[cols="1,2"]
|===
|CGI |役割

|`list-resource.cgi`
|manifest backed resource を日付条件で一覧します。

|`search-resource.cgi`
|日付範囲とキーワードで resource を検索します。

|`find-resource.cgi`
|`search-resource.cgi` と同じ検索仕様の互換入口です。

|`load-file.cgi`
|サムネイル、プレビュー、元ファイルを認可付きで配信します。

|`update-resource.cgi`
|コメント等、resource に付随する編集情報を更新します。
|===

`find-resource.cgi` と `search-resource.cgi` は同じ条件体系に統合します。

== ノートへの追加

選択したコンテンツをノートへ追加する場合、ファイルはコピーしません。

ノート側には次を追加します。

* `resources[]` にアップロードコンテンツへの参照
* `pages[].nodes[]` に表示ノード
* ノードから `resources[]` への `resourceRef`

参照先は常に `area=upload` の manifest backed resource です。

== 新規ノート作成

選択したコンテンツだけを含む新規ノートを作成できます。

この場合も、ファイル本体は `upload/` に置いたまま、ノート JSON には参照だけを保存します。

== 旧形式データ

次のデータはホーム画面には表示しません。

* `manifest.json` がないアップロードディレクトリ
* `resource.json` のみの旧リソース
* `note/new_note/resource/` にだけ存在するサムネイル
* 途中で保存に失敗した空ディレクトリ

今後の実装は、旧形式救済よりも manifest backed resource の整合性を優先します。
