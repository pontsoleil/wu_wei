== WuWei ホーム画面仕様

この文書は、WuWei ホーム画面の現行仕様を定義する。
ホーム画面は Resource library を一覧し、選択した Resource を現在編集中のノートへ追加、または選択 Resource だけを含む新規ノートを作成する入口である。

== 正本データ

ホーム画面と Resource 検索の正本は、次の `resource.json` である。

[source,text]
----
data/{user_uuid}/resource/YYYY/MM/DD/{resource_uuid}/resource.json
----

`upload/YYYY/MM/DD/{upload_uuid}/manifest.json` はアップロード実体の管理用マニフェストであり、ホーム一覧の主入力ではない。
`resource.json` は `storage.files[]` と `storage.manifest` により、実体ファイル、サムネイル、プレビュー PDF を `upload/` 側へ参照する。

== Resource card 生成

`list-resource.cgi` と `search-resource.cgi` は `resource/**/resource.json` を走査し、ホーム画面用の card JSON に変換する。

主な変換は次の通りである。

[cols="1,3", options="header"]
|===
|カード項目 |取得元

|`id`
|`resource.id`。未定義の場合は resource ディレクトリ名。

|`label`, `name`
|`resource.label`、`resource.identity.title`、`resource.id` の順に使用する。

|`description`
|`resource.description`。

|`contenttype`
|`resource.media.mimeType`、`resource.mimeType`、`storage.files[role=original].mimeType`。

|`thumbnail_url`
|`storage.files[role=thumbnail]` から `load-file` URL を生成する。

|`preview_url`
|`storage.files[role=preview]` があればそれを使用する。なければ original または外部 URI を使用する。

|`download_url`
|`storage.files[role=original]` または外部 URI。

|`value.resource`
|Content node に埋め込む Resource 参照。
|===

== ファイル参照方式

実体ファイルの配信 URL は、常に `area + path + role` 方式で生成する。

[source,text]
----
/wu_wei2/server/load-file.cgi?area=upload&path=YYYY/MM/DD/{upload_uuid}/thumbnail.jpg&role=thumbnail
----

`resource_id` を直接指定してファイルを解決する方式は使用しない。
`resource.json` には OS 絶対パスや公開 URL を保存せず、論理パスを保存する。

== 検索条件

ホーム画面では、次の条件で `resource.json` を絞り込む。

[cols="1,2", options="header"]
|===
|条件 |意味

|`year`, `month`
|登録年月で絞り込む。

|`date`
|登録日で絞り込む。

|`start_date`, `end_date`
|登録日の範囲で絞り込む。

|`term`
|Resource の title、label、URI、説明、ファイル名などを対象にしたキーワード検索。
|===

登録日は、原則として `resource/YYYY/MM/DD/` のパス日付、または `resource.audit.createdAt` / `resource.audit.lastModifiedAt` から判断する。

== CGI

[cols="1,3", options="header"]
|===
|CGI |役割

|`list-resource.cgi`
|`resource/**/resource.json` を日付条件で一覧する。

|`search-resource.cgi`
|`list-resource.cgi` と同じ実装を使用し、日付範囲とキーワードで Resource を検索する。

|`load-file.cgi`
|`area`、`path`、`role` を検証し、認可済みファイルを配信する。

|`update-resource.cgi`
|コメント、説明、ホーム表示メタデータなど Resource JSON の編集情報を更新する。
|===

== ノートへの追加

ホーム画面から Resource をノートへ追加する場合、アップロード済みファイル本体はコピーしない。
ノート JSON の Content node には、Resource の compact 定義と `storage.files[]` の参照を保存する。

== 表示対象外

次のデータはホーム画面の Resource library には表示しない。

* `resource.json` が存在しないディレクトリ
* 不正な JSON の `resource.json`
* `home.hidden` または `hiddenFromHome` が true の Resource
* `note/new_note/resource/` に一時的に残った旧スナップショット

== Office 文書と preview

Office 文書は、アップロード時に `preview.pdf` が生成される場合がある。
ホーム画面、情報ペイン、Contents axis は `storage.files[role=preview]` を参照する。
旧データの `pdf-preview` role は読み込み互換だけで扱い、新規生成では `preview` に統一する。
