以下のように、**モジュールの責務**と**画面設計の対象**を分けて整理する。

中心となる考え方は、次です。

[source,text]
----
upload / remote は「取得元・保管方式」
video / document / image は「コンテンツ種別」
timeline / contents は「コンテンツに対する視点軸」
edit / info は「対象の役割に応じた画面」
----

'''''

== 1. 全体分類

=== 1. resource source：取得元

これは物理属性の違いです。

[width="100%",cols="1,4,8",options="header",]
|===
^|source ^|意味 ^|主な属性

|defined |ノートキャンバスに定義した要素 |Topic, Memo, Generic group (vertical, horizontal, simple), Timeline, Contents, Representative Node

|remote |ネット上の URL |URL, title, site label, thumbnail

|upload |アップロードされたファイル |remote の属性に加えて、 +
file size, upload date, original filename, storage path, mimeType, preview path

|generated |変換生成物 |PDF preview, thumbnail, preview image

|embedded |将来拡張用 |data URL 等
|===

ここで重要なのは、**upload は UI の主分類にしない**ことです。 `+upload+`
はあくまで resource metadata です。

'''''

=== 2. resource kind：コンテンツ種別

これはユーザーが扱う対象の種類です。

[source,text]
----
document
  documentKind: 文書はさらに分けます。
    pdf
    office
    html
    text
video
  videoKind: 映像も分けます。
    youtube
    vimeo
    mp4
    mp3
image
audio
web
other
----

たとえば次のように分類します。

[cols="1,1,1,2",options="header",]
|===
^|kind ^|source ^|sub kind ^|表示・編集
|document |upload |office |document viewer / converted PDF
|document |remote |html |iframe / browser
|video |remote |youtube |video player
|video |upload |mp4 |video player
|image |upload |image |image viewer
|===

'''''

=== 3. perspective axis：視点軸

Timeline / Contents は、単なる video / document
の機能ではなく、**同じコンテンツに複数の視点を与えるための軸**です。

[source,text]
----
Content (video) node
  ├─ Timeline perspective A
  └─ Timeline perspective B
----

[source,text]
----
Content (document) node
  ├─ Contents perspective C
  └─ Contents perspective D
----

同じ動画でも、発表構成、技術論点、引用対象など、複数の Timeline 軸を定義できます。 +
同じ文書でも、章立て、技術論点、コメント対象、引用候補など、複数の Contents 軸を定義できます。

'''''

== 4. メニュー判定の優先順位

編集・情報表示の振り分けは、次の優先順位がよいです。

=== info

[source,text]
----
1. Timeline representative / axis / segment
   -> info-timeline

2. Contents representative / axis / PageMarker
   -> info-contents

3. video Content
   -> info-video

4. document Content
   -> info-document

5. image Content
   -> info-image

6. other Content
   -> info-resource or info-generic

7. Topic / Memo / Link / generic group
   -> existing generic info
----

=== edit

[source,text]
----
1. Timeline representative / axis / segment
   -> edit-timeline

2. Contents representative / axis / PageMarker
   -> edit-contents

3. video Content
   -> edit-video

4. document Content
   -> edit-document

5. image Content
   -> edit-image

6. other Content
   -> edit-resource or edit-generic

7. Topic / Memo / Link / generic group
   -> existing generic edit
----

この順序が重要です。 `+upload+` を先に判定すると、upload された mp4 や
docx が `+edit-uploaded+` に落ちてしまいます。

'''''

== 9. 共通 style 定義

=== 1. 位置づけ

`wuwei.style.js` は、ノートキャンバス上に表示される対象の見た目を扱う共通モジュールとする。
ここでいう style は、取得元が upload か remote か、対象が document か video かといったコンテンツ種別とは独立した、画面上の外観属性である。

[source,text]
----
style 領域
  ノートキャンバス上の見た目を扱う
  shape / size / fill / line / font / label align / label offset

content-kind 領域
  コンテンツ種別ごとの意味的属性を扱う
  document / video / image / resource metadata

perspective 領域
  Timeline / Contents の視点軸属性を扱う
  axis direction / length / width / color / page offset / segment / PageMarker
----

したがって、`wuwei.style.js` は画面用共通処理とデータ用共通処理に分けず、style 領域を生成・初期化・読取・反映する共通モジュールとして定義する。

=== 2. 対象とする style 属性

[cols="24%,76%",options="header"]
|===
|分類 |属性
|基本表示 |`label`, `description`, `visible`, `opacity`
|形状 |`shape`, `size.width`, `size.height`, `size.radius`
|塗り |`style.fill`, `color`
|線 |`style.line.kind`, `style.line.color`, `style.line.width`, `outline`
|フォント |`style.font.family`, `style.font.size`, `style.font.color`, `style.font.align`
|ラベル配置 |`labelAlign`, `labelOffset.x`, `labelOffset.y`, `axisPos`
|リンク |`shape`, `routing.path`, `style.line`, `style.font`, `label`, `x`, `y`
|軸 |`spine.visible`, `spine.color`, `spine.width`, `length`, `orientation`
|===

style 領域は、通常ノード、代表ノード、Timeline segment、Contents PageMarker、Link、軸の外観に共通して利用する。
ただし、軸の意味的属性、例えば Timeline の `timeStart` / `timeEnd`、Contents の `pageOffset` / `pageNumber` / `anchorHref` は、それぞれ `wuwei.timeline.js` / `wuwei.contents.js` の責務である。

=== 3. style profile

全対象に同じ項目を表示するのではなく、対象の役割に応じて style profile を使い分ける。

[cols="24%,76%",options="header"]
|===
|profile |主な対象
|`normalNode` |通常 Topic / Content / Memo
|`contentNode` |document / video / image などの Content 本体
|`representativeNode` |generic group / timeline / contents の代表ノード
|`timelineSegment` |Timeline の start / point / end segment
|`contentsPageMarker` |Contents の PageMarker
|`link` |通常 Link
|`axis` |horizontal / vertical group spine、Timeline axis、Contents axis
|===

=== 4. `wuwei.style.js` の責務

[source,javascript]
----
wuwei.style.getProfile(target, option)
wuwei.style.renderEditSection(target, option)
wuwei.style.renderInfoSection(target, option)
wuwei.style.bindEditSection(root, target, option)
wuwei.style.readEditValues(root, option)
wuwei.style.applyEditValues(target, values, option)
wuwei.style.normalize(target, option)
----

`renderEditSection()` は、style 領域の HTML を返す。
`bindEditSection()` は、align アイコン、color picker、size input、textarea rows などの style 領域の UI を初期化する。
`applyEditValues()` は、対象 node / link / group に style 値を反映する。

=== 5. `wuwei.menu.js` との関係

`wuwei.menu.js` は style 処理の本体を持たない。
右クリック対象を判定し、どの edit / info domain のどの section を開くかを指定するだけにする。

[source,javascript]
----
wuwei.edit.open(target, {
  domain: 'contents',
  section: 'style'
});

wuwei.edit.open(target, {
  domain: 'contents',
  section: 'axis'
});
----

== 10. 対象データモデル定義

=== 1. `current.page` と `graph` の役割

WuWei app2 では、`current.page` と `graph` を明確に区別する。

[cols="20%,80%",options="header"]
|===
|領域 |役割
|`current.page` |永続化対象の正本。保存・読込・undo / redo の基準となる。`nodes`, `links`, `groups`, `transform`, `thumbnail` などを保持する。
|`graph` |描画・操作・simulation のための runtime 表現。`current.page` から生成され、pseudo link / pseudo group overlay / force simulation 用属性などを含むことがある。
|===

基本方針は次である。

[source,text]
----
current.page
  durable model
  保存対象
  node / link / group の正本

graph
  runtime render model
  描画・ドラッグ・simulation 用
  pseudo link / pseudo overlay / runtime fields を含む
----

`model.setGraphFromCurrentPage()` は `current.page` を graph に展開する。
`current.page` は正本であり、graph は runtime 展開である。real node / real link は原則として `current.page.nodes` / `current.page.links` の同一オブジェクト参照を graph 側でも扱うため、graph から `current.page` へ変換し直す専用関数は定義しない。
ただし、pseudo link や pseudo overlay などの描画補助要素は保存対象にしない。

=== 2. `current.page` の基本構造

[source,javascript]
----
current.page = {
  id: '...',
  pp: 1,
  name: '...',
  description: '...',

  transform: {
    x: 0,
    y: 0,
    scale: 1
  },

  nodes: [],
  links: [],
  groups: [],

  thumbnail: '...'
};
----

`transform` は、ノートキャンバス全体の pan / zoom 状態を表す。
SVG 描画では、この transform をキャンバスに適用するため、node の座標と画面上の見え方は区別して扱う。

=== 3. `graph` の基本構造

[source,javascript]
----
graph = {
  nodes: [],
  links: [],
  mode: 'draw' | 'view' | 'simulation',
  ...
};
----

graph には、`current.page.nodes` / `current.page.links` から来る real record に加え、次のような runtime 要素が含まれる。

[cols="28%,72%",options="header"]
|===
|runtime 要素 |説明
|pseudo axis link |horizontal / vertical group、Timeline axis、Contents axis を描画するための仮想 Link
|pseudo group overlay |simple group の外周や操作入口として使う仮想表示要素
|force simulation fields |`vx`, `vy`, `fx`, `fy`, `dragging`, `changed`, `filterout` など
|computed routing |Link path や制御点など、描画時に補助的に使う値
|===

=== 4. Node 共通定義

==== `current.page.nodes` における Node

[source,javascript]
----
{
  id: '...',
  type: 'Topic' | 'Content' | 'Memo' | 'Segment' | 'PageMarker',
  label: '...', // Memo では使用しない
  description: {
    format: 'asciidoc' | 'plain' | 'plain/text',
    body: '...'
  },

  x: 0,
  y: 0,
  visible: true,

  shape: 'RECTANGLE' | 'CIRCLE' | 'MEMO' | '...',
  size: {
    width: 100,
    height: 40,
    radius: 20
  },

  style: {
    fill: '#ffffff',
    font: {
      family: 'sans-serif',
      size: 12,
      color: '#333333',
      align: 'center'
    },
    line: {
      kind: 'SOLID',
      color: '#808080',
      width: 1
    }
  },

  audit: {
    createdBy: '...',
    createdAt: '...',
    lastModifiedBy: '...',
    lastModifiedAt: '...'
  }
}
----

==== graph における Node

graph 上の Node は、原則として `current.page.nodes` の real node である。
ただし、描画・ドラッグ・simulation のために次の runtime 属性を持つことがある。

[source,javascript]
----
{
  ...node,
  vx: 0,
  vy: 0,
  fx: null,
  fy: null,
  dragging: false,
  changed: true,
  filterout: false
}
----

これら runtime 属性は、永続データの意味を持たない。
保存時には、正本である `current.page` から pseudo node / pseudo link などの runtime 要素を含まない real data を出力する。

=== 5. Node 種別ごとの定義

[cols="18%,32%,50%",options="header"]
|===
|種別 |`current.page` での定義 |graph での扱い

|Topic
|`type: 'Topic'`。通常の概念ノード。`label`, `description`, `shape`, `size`, `style`, `x`, `y` を持つ。
|real node として描画される。simulation / drag により runtime 座標や force 属性を持つことがある。

|Content
|`type: 'Content'`。`resource` を持つ。`resource` により upload / remote、document / video / image 等を判定する。legacy な `uri`, `url`, `format`, `download_url` は Node 直下に持たせない方針。
|real node として描画される。Content 本体の edit / info は resource kind により document / video / image / resource に振り分ける。

|Memo
|`type: 'Memo'`。メモ本文を `description.body` に持つ。付箋紙風の `shape` / `style.fill` を持つ。
|real node として描画される。Memo 固有の見た目は style profile で扱う。

Memo は `label` を持たない。表示本文は `description.body` として定義する。
旧データ又は共通 Node 正規化処理等により Memo に `label` が存在する場合でも、読み込み時又は保存時に不要項目として無視する。
Memo の `style.label` も v2 データモデル及び保存対象には含めない。

|Segment
|`type: 'Segment'`。Timeline の member node。`groupRef`, `topicKind: 'timeline-point'`, `axisRole`, `mediaStart`, `mediaEnd`, `playDuration` を持つ。
|Timeline axis 上に配置される real node。drag により `mediaStart` / `mediaEnd` または `axisPos` が更新されることがある。

|PageMarker
|`type: 'PageMarker'`。Contents の member node。`groupRef`, `topicKind: 'contents-page'`, `documentRef`, `pageNumber`, `anchorHref`, `htmlAnchorHref` を持つ。
|Contents axis 上に配置される real node。PDF / Office では pageNumber 順、HTML では anchorHref を参照して開く。
|===

=== 6. Content node と resource

Content node では、物理属性と意味的分類を `resource` に持たせる。

[source,javascript]
----
{
  id: '...',
  type: 'Content',
  label: '...',
  description: {...},
  resource: {
    id: '...',
    source: 'upload' | 'remote' | 'generated' | 'embedded',
    kind: 'document' | 'video' | 'image' | 'audio' | 'other',
    documentKind: 'pdf' | 'office' | 'html' | 'text',
    videoKind: 'youtube' | 'vimeo' | 'mp4' | 'mp3',
    canonicalUri: '...',
    uri: '...',
    title: '...',
    mimeType: '...',
    storage: {
      files: [
        { role: 'original', path: '...', mimeType: '...' },
        { role: 'preview', path: '...' },
        { role: 'thumbnail', path: '...' }
      ]
    }
  }
}
----

`mimeType` は参考情報として保持・表示するが、処理分岐の主キーにはしない。
文書・映像の判定は、原則として拡張子、URL、`resource.kind` / `resource.documentKind` / `resource.videoKind` などに基づく。

Group は `current.page.groups` に保持する。
Group 自体は原則として Node ではない。
ただし graph 上では、操作や描画のために pseudo node / pseudo link / overlay として現れることがある。

[source,javascript]
----
{
  id: '...',
  name: '...',
  type: 'simple' | 'horizontal' | 'vertical' | 'timeline' | 'contents',
  groupType: 'axis' | undefined,
  description: {
    format: 'plain' | 'asciidoc',
    body: '...'
  },
  visible: true,
  moveTogether: true,
  orientation: 'auto' | 'horizontal' | 'vertical',
  spine: {
    kind: 'SOLID' | 'DASHED' | '...',

    paddingTop: 12,
    paddingRight: 12,
    paddingBottom: 12,
    paddingLeft: 12
  },

    { nodeId: '...', order: 1, role: 'member', value: '', offset: 0 }

  representativeNodeId: '...',
  targetNodeId: '...',
  parentGroupId: '',
  hierarchical: false,
  axis: {...},
  length: 480,
  audit: {...}
}
----

=== 8. Group 種別ごとの定義

[cols="16%,42%,42%",options="header"]
|===
|種別 |`current.page` での定義 |graph での扱い

|simple group
|`current.page.groups[]` に `type: 'simple'` として保持する。`members` は real node の `nodeId` を参照する。代表ノードを持つ場合は `representativeNodeId` で参照する。
|外周 box / overlay は graph 上で描画補助として扱う。group 自体を real node として保存しない。

|horizontal group
|`type: 'horizontal'`。`orientation: 'horizontal'`。member node の y 座標を中心に揃える group。spine を持つ。
|graph では spine を pseudo link として描画する。member node と representative node は real node。

|vertical group
|`type: 'vertical'`。`orientation: 'vertical'`。member node の x 座標を中心に揃える group。spine を持つ。
|graph では spine を pseudo link として描画する。member node と representative node は real node。

|timeline group
|`type: 'timeline'`, `groupType: 'axis'`。video Content を `targetNodeId` / `mediaRef` で参照する。`hierarchical: true` とし、Content に付属する semantic sub-group として扱う。
|graph では Timeline axis を pseudo link として描画する。Segment node と代表ノードは real node。

|contents group
|`type: 'contents'`, `groupType: 'axis'`。document Content を `targetNodeId` / `documentRef` で参照する。`hierarchical: true` とし、Content に付属する semantic sub-group として扱う。
|graph では Contents axis を pseudo link として描画する。PageMarker node と代表ノードは real node。
|===

=== 9. Representative node

Representative node は、Group を代表する real Topic node である。
`current.page.nodes` に保存されるが、`group.members` には含めない。
Group とは `groupRef` / `groupRole` / `representativeOf` / `representativeNodeId` により関連付ける。

[source,javascript]
----
{
  id: '...',
  type: 'Topic',
  label: 'Contents' | 'Timeline' | '...',
  description: {...},
  groupRef: 'group-id',
  groupRole: 'representative',
  representativeOf: {
    kind: 'group',
    id: 'group-id'
  },
  topicKind: 'group-representative' | 'timeline-representative' | 'contents-representative',
  representativeType: 'simple' | 'horizontal' | 'vertical' | 'timeline' | 'contents',
  axisPos: 0,
  shape: 'RECTANGLE',
  size: {...},
  style: {...},
  x: 0,
  y: 0
}
----

Representative node の edit / info は、次の2種類の属性を同時に扱う。

[source,text]
----
代表ノード自身の外観
  shape / size / fill / line / font / label align / offset

代表している group / perspective axis の属性
  orientation / length / spine / page offset / time range
----

このため、代表ノードの編集画面では `style` section と `axis` section を分ける。

== 結論

最も重要な整理は次です。

[source,text]
----
upload は UI 分類ではなく resource metadata。
video / document / image が Content 本体の UI 分類。
timeline / contents は Content に対する Perspective axis。
代表ノードは Perspective axis の操作入口。
----

そのため、画面分類は次のようにするのが妥当です。

[source,text]
----
Content 本体:
  edit-document / info-document
  edit-video / info-video
  edit-image / info-image

視点軸:
  edit-timeline / info-timeline
  edit-contents / info-contents

物理属性:
  resource metadata section

fallback:
  edit-resource / info-resource
----

この整理にすると、upload 文書が `+edit-uploaded+` に落ちる、upload
動画が `+edit-uploaded+` に落ちる、PageMarker が `+info-uploaded+`
に展開される、といった混乱を避けられます。
