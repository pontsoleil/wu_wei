= WuWei CGI 役割仕様
:toc:
:toclevels: 3
:sectnums:

この文書は、ノートとリソースを扱う CGI の役割分担を定義します。

== 基本方針

CGI はシェルスクリプトを中心に保守します。

通常のノート保存では、アップロード済みファイルをノート配下へコピーしません。保存するのはノート JSON と参照です。

== CGI 一覧

[cols="1,2"]
|===
|CGI |役割

|`upload.cgi`
|アップロードファイルを `upload/YYYY/MM/DD/{upload_uuid}/` に保存し、`manifest.json`、`thumbnail.jpg`、必要に応じて `preview.pdf` を作成します。

|`list-resource.cgi`
|`upload/**/manifest.json` を読み、ホーム画面用の一覧 JSON を返します。

|`search-resource.cgi`
|日付範囲とキーワードで `upload/**/manifest.json` を検索します。

|`find-resource.cgi`
|`search-resource.cgi` と同じ仕様の互換入口です。

|`update-resource.cgi`
|コメント、説明、表示用メタデータを更新します。ファイル本体は移動しません。

|`save-note.cgi`
|ノート JSON を `note/YYYY/MM/DD/{note_uuid}/note.json` に保存します。resource は参照保存です。

|`load-note.cgi`
|保存済みノート JSON を読み込みます。

|`list-note.cgi`
|`note.json` が存在する保存済みノートだけを一覧します。`ERROR NOTE NOT FOUND` になるディレクトリは表示しません。

|`remove-note.cgi`
|ノートを削除または trash へ移動します。参照先 upload resource は削除しません。

|`export-note.cgi`
|ノート JSON が参照する upload resource を収集し、ZIP 化します。

|`import-note.cgi`
|ZIP からノート JSON と upload resource を復元します。

|`load-file.cgi`
|`area` と `path` を検証し、認可済みファイルを配信します。
|===

== upload.cgi

`upload.cgi` はアップロードコンテンツの正本ディレクトリを作成します。

[source,text]
----
data/{user_uuid}/upload/YYYY/MM/DD/{upload_uuid}/
  manifest.json
  original-file.ext
  thumbnail.jpg
  preview.pdf
----

`manifest.json` には元ファイルの SHA-256 を `original.sha256` として保存します。

Office 文書では LibreOffice を使って `preview.pdf` を生成し、その先頭ページから `thumbnail.jpg` を作成します。

== save-note.cgi

`save-note.cgi` はノート保存の責務だけを持ちます。

保存対象は次の通りです。

* ノート ID
* ノート名
* 説明
* ページ
* グループ
* ノード
* リンク
* resource 参照
* ノートサムネイル

アップロードファイル本体、アップロードサムネイル、Office preview PDF はコピーしません。

== list-resource.cgi / search-resource.cgi

リソース一覧は manifest backed resource だけを返します。

[source,text]
----
data/{user_uuid}/upload/YYYY/MM/DD/{upload_uuid}/manifest.json
----

旧形式の `resource.json` だけを持つデータは返しません。

== list-note.cgi

ノート一覧は `note.json` が存在し、読み込み可能なノートだけを返します。

空ディレクトリ、保存失敗ディレクトリ、`new_note` は通常一覧に含めません。

救済表示が必要な場合は、明示的なオプションでのみ `new_note` を対象にします。

== export-note.cgi

ノートダウンロードでは、次を ZIP 化します。

* `note/YYYY/MM/DD/{note_uuid}/note.json`
* ノートサムネイル
* `note.json` が参照する `upload/YYYY/MM/DD/{upload_uuid}/` ディレクトリ

通常保存時にコピーしない代わりに、ダウンロード時に参照先を収集します。

== import-note.cgi

ノートアップロードでは、ZIP 内の upload resource を復元してから note JSON を保存します。

同じ SHA-256 の upload resource が既に存在する場合は、既存ディレクトリを再利用できます。

== load-file.cgi

`load-file.cgi` は、ブラウザから `data/` へ直接アクセスさせないための認可付き配信入口です。

指定形式は次の通りです。

[source,text]
----
load-file.cgi?area=upload&path=YYYY/MM/DD/{upload_uuid}/thumbnail.jpg&user_id={user_uuid}
----

`area` は許可された値だけを受け付けます。

* `upload`
* `note`
* `thumbnail`
* `resource`（旧互換が必要な場合のみ）

Linux/nginx では `X-Accel-Redirect` により内部パスから配信します。
