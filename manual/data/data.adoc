= WuWei データ仕様
:toc:
:toclevels: 3
:sectnums:

この文書は、2026-05-01 時点の WuWei の保存データ仕様を定義します。

== 基本方針

WuWei のデータは、次の考え方で単純化します。

* アップロードされたファイル本体、サムネイル、Office 変換 PDF は `upload/` を正本とします。
* ノート保存ではアップロード済みファイルをコピーせず、ノート JSON に参照を保存します。
* ノートのダウンロード時に、ノート JSON が参照するアップロードディレクトリを収集して ZIP 化します。
* 開発途中の旧形式データは一覧表示の対象外とします。
* 保存 JSON には OS の絶対パスを保存しません。`data/{user_uuid}` 配下からの相対パス、`area`、`role` を保存します。

== ディレクトリ構成

[source,text]
----
data/{user_uuid}/
  upload/YYYY/MM/DD/{upload_uuid}/
    manifest.json
    {original_file}
    thumbnail.jpg
    preview.pdf

  note/YYYY/MM/DD/{note_uuid}/
    note.json
    thumbnail.jpg

  trash/
----

`{upload_uuid}` と `{note_uuid}` は先頭に `_` を付けた UUID 文字列です。

== アップロードコンテンツ

アップロードコンテンツは `upload/YYYY/MM/DD/{upload_uuid}/manifest.json` を正本メタデータとします。

[source,json]
----
{
  "id": "_c2908f10-e2bb-4935-8655-275ccc9f3735",
  "type": "UploadResource",
  "version": 1,
  "created_at": "2026-05-01T15:51:00+0900",
  "created_by": "2f754971-e1b3-436c-9148-29c4a18a100f",
  "title": "XBRL GL Next Vision Statement",
  "kind": "document",
  "original": {
    "file": "XBRL_GL_Next_Vision_Statement_2026-04-02.docx",
    "display_name": "XBRL GL Next Vision Statement",
    "mime": "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
    "size": 41725,
    "sha256": "3ecc93c8e79291ad90a62ed20ba81df02b74f66ca51402413499c19d0b9dce8d"
  },
  "thumbnail": {
    "file": "thumbnail.jpg",
    "mime": "image/jpeg",
    "size": 10998,
    "sha256": "1b05dfb943af9037796fbb990d7d813b4a796e69360221d6a0f78f8bf4c79ff7",
    "display_size": "141x200"
  },
  "preview": {
    "file": "preview.pdf",
    "mime": "application/pdf",
    "size": 46918,
    "sha256": "c2b6e773df6097004bfac3007264fa5c82001e396b6c126b0f8e4bbb01fdc18c",
    "generated_by": "libreoffice"
  }
}
----

=== SHA-256

`original.sha256` はアップロードされた元ファイル本体の SHA-256 です。

同じ `sha256` のファイルは同一コンテンツとみなし、原則として重複保存しません。再アップロード時は既存の `upload_uuid` を再利用できます。

`thumbnail.sha256` と `preview.sha256` は、派生ファイルの検証用です。重複判定の主キーには使いません。

=== サムネイル

`thumbnail.jpg` はアップロードコンテンツの表示用サムネイルです。

* PDF は先頭ページから生成します。
* 画像は画像本体から生成します。
* Office 文書は LibreOffice で `preview.pdf` を生成し、その先頭ページから `thumbnail.jpg` を生成します。
* 生成できない場合は UI 側でアイコン表示にフォールバックします。

== ノート

ノートは `note/YYYY/MM/DD/{note_uuid}/note.json` を正本とします。

ノート JSON はグループ、ノード、リンク、ノート自身のメタデータを保存します。アップロードコンテンツのファイル本体は保存しません。

[source,json]
----
{
  "note_id": "_925f3c6a-8fc1-4565-9d59-e4d4242ff1d1",
  "note_name": "test20260501",
  "description": "",
  "currentPage": "_999ffe90-6536-425a-a63c-b2c36cd0126a",
  "resources": [
    {
      "id": "_c2908f10-e2bb-4935-8655-275ccc9f3735",
      "kind": "document",
      "title": "XBRL GL Next Vision Statement",
      "area": "upload",
      "base": "2026/05/01/_c2908f10-e2bb-4935-8655-275ccc9f3735",
      "manifest": "2026/05/01/_c2908f10-e2bb-4935-8655-275ccc9f3735/manifest.json",
      "original": "2026/05/01/_c2908f10-e2bb-4935-8655-275ccc9f3735/XBRL_GL_Next_Vision_Statement_2026-04-02.docx",
      "thumbnail": "2026/05/01/_c2908f10-e2bb-4935-8655-275ccc9f3735/thumbnail.jpg",
      "preview": "2026/05/01/_c2908f10-e2bb-4935-8655-275ccc9f3735/preview.pdf"
    }
  ],
  "pages": []
}
----

ノードは `resourceRef` または同等の `resource.id` で `resources[]` のエントリを参照します。

== 未保存ノート

新規ノートの未保存状態では、ブラウザ上のノート ID と一時保存先として固定値 `new_note` を使えます。

ただし、アップロードコンテンツ本体、サムネイル、プレビュー PDF は `note/new_note/resource/` に置かず、常に `upload/YYYY/MM/DD/{upload_uuid}/` に置きます。

保存時に新しい `{note_uuid}` を発行し、ノート JSON 内の `note_id` を更新します。アップロードコンテンツへの参照は `upload/` のまま維持します。

== ホーム画面とリソース検索

ホーム画面、`list-resource.cgi`、`search-resource.cgi`、`find-resource.cgi` は `upload/**/manifest.json` のみをリソース一覧の入力とします。

旧形式の `resource/YYYY/MM/DD/{resource_uuid}/resource.json` だけを持つデータは、今後の一覧には表示しません。

検索条件は次の通りです。

* `year`
* `month`
* `date`
* `start_date`
* `end_date`
* `term`

`term` はタイトル、元ファイル名、コメント等の検索に使います。

== ファイル配信

ブラウザから `data/` 配下へ直接アクセスさせません。

ファイル表示は `load-file.cgi` に `area` と `path` を渡して行います。

[source,text]
----
/wu_wei2/server/load-file.cgi?area=upload&path=2026%2F05%2F01%2F_c2908f10...%2Fthumbnail.jpg&user_id=...
----

Linux/nginx 環境では `X-Accel-Redirect` を使い、認可確認後に内部パスから配信します。

== ノートのダウンロードとアップロード

ノート保存とノートダウンロードは分けて考えます。

* ノート保存: `note.json` に参照を保存します。
* ノートダウンロード: `note.json` が参照する `upload/` ディレクトリを収集して ZIP 化します。
* ノートアップロード: ZIP 内の `upload/` ディレクトリを復元し、`note.json` の参照を復元先に合わせます。

通常保存時にアップロードコンテンツをノートディレクトリへコピーしないことで、保存処理を軽く保ちます。

== 旧データの扱い

開発途中で作成された次のようなデータは、整合性が保証できないため一覧表示から除外します。

* `manifest.json` を持たないアップロードディレクトリ
* `resource.json` だけを持つ旧リソースディレクトリ
* 空のノートディレクトリ
* `note.json` が存在しないノートディレクトリ
* `new_note` のまま保存完了していない作業ディレクトリ

必要な場合のみ手作業で移行します。通常のコードでは旧形式救済のための複雑な分岐を増やしません。
